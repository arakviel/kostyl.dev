# Material Creator Skill

Цей скіл призначений для генерації глибокого, цікавого та інтерактивного навчального контенту українською мовою.

## Філософія та Принципи

### 1. Engagement First (Залученість передусім)
- Починайте з проблеми або реального сценарію (Storytelling). Читач має розуміти *навіщо* йому це потрібно.
- Використовуйте живий, але академічно точний стиль.

### 2. The "Why" Before The "How"
- Ніколи не вводьте термін чи метод без пояснення проблеми, яку він вирішує.
- Показуйте "еволюцію думки": чому ми обрали саме це рішення.

### 3. Text First (Текст важливіший за код)
- Стаття — це розгорнуте текстове пояснення, де код лише підкріплює думку.
- Обсяг тексту має бути значним: на 30 рядків коду — мінімум 30 рядків пояснень.

### 4. Code Anatomy (Анатомія Коду)
Кожен нетривіальний блок коду має супроводжуватися розбором:
1. **Контекст**: Що ми зараз реалізуємо?
2. **Код**: Блок коду з підсвічуванням/focus (MDC).
3. **Декомпозиція**: Текстове пояснення ключових частин після коду.

### 5. No Silent Code
Заборонено вставляти код без пояснення "магії", що відбувається всередині.

## Інструментарій (Docus Components)

Використовуйте компоненти Docus для інтерактивності. Систематизуйте контент за допомогою:
- **Callouts**: `::note`, `::tip`, `::warning`, `::caution` для акцентів.
- **Navigation**: `::tabs`, `::steps` для покрокових інструкцій.
- **Visualization**: `::mermaid`, `::plant-uml` для діаграм.
- **Live Preview**: `::html-preview` для живого відображення HTML/CSS.

*Повний довідник компонентів доступний у `resources/DOCUS_COMPONENTS.md`.*

## Уніфікована Структура Матеріалу

1. **Вступ**: Проблема, ціль (Goal), Prerequisites.
2. **Фундаментальні Концепції**: Точні визначення, аналогії.
3. **Архітектура**: "Under the Hood", Lifecycle, діаграми.
4. **Практична Реалізація**: `::steps`, приклади коду, Verification.
5. **Порівняння**: Порівняльні таблиці, Use Cases.
6. **Практика**: Завдання 3 рівнів складності.
7. **Резюме**: Ключові думки.

## Правила Мови
- **Українська мова** — основна.
- **Англійські терміни** в дужках при першому вживанні.
- **Стандартні англіцизми** (framework, middleware) залишати без перекладу, якщо це доцільно.
