# Інтеграційне тестування з WebApplicationFactory

## 1. Межа між Unit та Integration тестами

У попередніх розділах ми ізолювали все зовнішнє через Moq та тестували логіку в «вакуумі». Це Unit тестування — швидке, просте, дає точне розуміння де зламалося.

Але unit-тести мають сліпу зону: вони **не перевіряють, як компоненти інтегруються один з одним**. Наприклад:

- Чи правильно налаштований middleware-ланцюжок?
- Чи `MapGet("/users/{id}")` насправді передає `id` у handler?
- Чи повертає API коректний HTTP-статус 404 при відсутньому ресурсі?
- Чи спрацьовує авторизація і блокує незареєстрованих користувачів?

Відповіді на ці питання дають **інтеграційні тести** — вони піднімають реальний (або максимально наближений до реального) стек застосунку і надсилають справжні HTTP-запити.

## 2. WebApplicationFactory — «Тестовий дублер» всього застосунку

`Microsoft.AspNetCore.Mvc.Testing.WebApplicationFactory<TEntryPoint>` — це клас зі стандартної бібліотеки Microsoft, який:

1. **Піднімає повноцінний ASP.NET Core сервер** у пам'яті (in-process), без реального мережевого порту.
2. Надає **`HttpClient`**, що з'єднаний з цим сервером напряму (через `TestServer`).
3. Дозволяє **замінити будь-які залежності** (наприклад, реальну БД на in-memory) через перевизначення DI-контейнера.

::premium-alert{type="info" title="Ключова перевага"}
Вам не потрібно запускати сервер окремо або вказувати URL. `WebApplicationFactory` запускає увесь ваш застосунок всередині тестового процесу. Це означає: **тест сам є клієнтом і сервером одночасно**.
::

## 3. Налаштування: Встановлення пакету

```bash
dotnet add MyApp.Tests package Microsoft.AspNetCore.Mvc.Testing
```

Переконайтеся, що у `MyApp.Tests.csproj` є правильний SDK і налаштування:

```xml [MyApp.Tests/MyApp.Tests.csproj] {5,12}
<Project Sdk="Microsoft.NET.Sdk">
  <PropertyGroup>
    <TargetFramework>net9.0</TargetFramework>
    <Nullable>enable</Nullable>
    <IsPackable>false</IsPackable>
  </PropertyGroup>

  <ItemGroup>
    <PackageReference Include="Microsoft.NET.Test.Sdk" Version="17.11.1" />
    <PackageReference Include="xunit" Version="2.9.2" />
    <PackageReference Include="xunit.runner.visualstudio" Version="2.8.2" />
    <PackageReference Include="Microsoft.AspNetCore.Mvc.Testing" Version="9.0.0" />
  </ItemGroup>

  <ItemGroup>
    <ProjectReference Include="..\MyApp\MyApp.csproj" />
  </ItemGroup>
</Project>
```

Також **обов'язково** додайте до `MyApp.csproj`:

```xml [MyApp/MyApp.csproj] {3}
<ItemGroup>
  <!-- Дозволяє тестовому проєкту бачити internal-класи -->
  <InternalsVisibleTo Include="MyApp.Tests" />
</ItemGroup>
```

## 4. Перший інтеграційний тест: Hello World

Нехай наш мінімальний API виглядає так:

```csharp [MyApp/Program.cs] showLineNumbers
var builder = WebApplication.CreateBuilder(args);
var app = builder.Build();

app.MapGet("/", () => "Hello from kostyl.dev!");
app.MapGet("/health", () => Results.Ok(new { Status = "Healthy" }));

app.Run();

// Цей рядок потрібен, щоб зробити Program доступним для WebApplicationFactory
public partial class Program { }
```

::caution
Рядок `public partial class Program { }` наприкінці `Program.cs` — обов'язковий. Він дозволяє `WebApplicationFactory<Program>` знайти точку входу застосунку. Без нього компілятор сховає клас `Program` і тест не зможе з ним працювати.
::

Тепер — перший інтеграційний тест:

```csharp [MyApp.Tests/Integration/HealthCheckTests.cs] showLineNumbers
using System.Net;

namespace MyApp.Tests.Integration;

public class HealthCheckTests(WebApplicationFactory<Program> factory)
    : IClassFixture<WebApplicationFactory<Program>>
{
    private readonly HttpClient _client = factory.CreateClient();

    [Fact]
    public async Task Get_HealthEndpoint_ReturnsOkStatus()
    {
        // Act — надсилаємо реальний GET-запит до тестового сервера
        var response = await _client.GetAsync("/health");

        // Assert
        Assert.Equal(HttpStatusCode.OK, response.StatusCode);
    }

    [Fact]
    public async Task Get_HealthEndpoint_ReturnsExpectedBody()
    {
        // Act
        var response = await _client.GetAsync("/health");
        var body = await response.Content.ReadAsStringAsync();

        // Assert
        Assert.Contains("Healthy", body);
    }

    [Fact]
    public async Task Get_UnknownEndpoint_Returns404()
    {
        var response = await _client.GetAsync("/does-not-exist");
        Assert.Equal(HttpStatusCode.NotFound, response.StatusCode);
    }
}
```

**Анатомія коду:**

- **Рядок 5**: Primary constructor із `WebApplicationFactory<Program>` — xUnit передає `factory` через механізм `IClassFixture`.
- **Рядок 6**: `IClassFixture<WebApplicationFactory<Program>>` — xUnit **разово** створює `WebApplicationFactory` для всього класу тестів і передає усім тестовим методам. Сервер піднімається **один раз** (не для кожного тесту окремо) → швидко.
- **Рядок 7**: `.CreateClient()` — повертає `HttpClient`, налаштований для роботи з тестовим сервером. Базова адреса автоматично встановлена на `http://localhost`.

## 5. Заміна залежностей: `ConfigureTestServices`

У реальних проєктах сервіси читають з реальних баз даних. Для тестів ми хочемо замінити їх на тестові версії. `WebApplicationFactory` дозволяє перевизначити DI-контейнер через `.WithWebHostBuilder()`:

Уявімо розширений API з репозиторієм:

```csharp [MyApp/Program.cs] showLineNumbers
var builder = WebApplication.CreateBuilder(args);
builder.Services.AddScoped<IProductRepository, ProductRepository>(); // Реальний репозиторій (PostgreSQL)
var app = builder.Build();

app.MapGet("/products/{id}", async (int id, IProductRepository repo) =>
{
    var product = await repo.GetByIdAsync(id);
    return product is null ? Results.NotFound() : Results.Ok(product);
});

app.Run();
public partial class Program { }
```

Тест, що замінює репозиторій на Moq:

```csharp [MyApp.Tests/Integration/ProductsApiTests.cs] showLineNumbers
using System.Net;
using Moq;
using Microsoft.Extensions.DependencyInjection;

namespace MyApp.Tests.Integration;

public class ProductsApiTests : IClassFixture<WebApplicationFactory<Program>>
{
    private readonly WebApplicationFactory<Program> _factory;

    public ProductsApiTests(WebApplicationFactory<Program> factory)
    {
        _factory = factory;
    }

    private HttpClient CreateClientWithMockedRepo(Mock<IProductRepository> repoMock)
    {
        return _factory.WithWebHostBuilder(builder =>
        {
            builder.ConfigureTestServices(services =>
            {
                // Видаляємо реальну реалізацію
                services.Remove(services.First(d =>
                    d.ServiceType == typeof(IProductRepository)));

                // Додаємо мок
                services.AddScoped(_ => repoMock.Object);
            });
        }).CreateClient();
    }

    [Fact]
    public async Task GetProduct_ExistingId_ReturnsOkWithProduct()
    {
        // Arrange
        var expectedProduct = new Product(1, "Gaming Laptop", 1299.99m);
        var repoMock = new Mock<IProductRepository>();
        repoMock.Setup(r => r.GetByIdAsync(1)).ReturnsAsync(expectedProduct);

        var client = CreateClientWithMockedRepo(repoMock);

        // Act
        var response = await client.GetAsync("/products/1");

        // Assert
        Assert.Equal(HttpStatusCode.OK, response.StatusCode);

        var product = await response.Content.ReadFromJsonAsync<Product>();
        Assert.NotNull(product);
        Assert.Equal("Gaming Laptop", product.Name);
        Assert.Equal(1299.99m, product.Price);
    }

    [Fact]
    public async Task GetProduct_NonExistingId_Returns404()
    {
        // Arrange
        var repoMock = new Mock<IProductRepository>();
        repoMock.Setup(r => r.GetByIdAsync(It.IsAny<int>())).ReturnsAsync((Product?)null);

        var client = CreateClientWithMockedRepo(repoMock);

        // Act
        var response = await client.GetAsync("/products/999");

        // Assert
        Assert.Equal(HttpStatusCode.NotFound, response.StatusCode);
    }
}
```

## 6. Кастомний `WebApplicationFactory`: Чистота в масштабі

Коли тестових класів більше двох — дублювати логіку налаштування у кожному `IClassFixture` нераціонально. Рішення — **кастомна фабрика**:

```csharp [MyApp.Tests/Infrastructure/CustomWebApplicationFactory.cs] showLineNumbers
namespace MyApp.Tests.Infrastructure;

public class CustomWebApplicationFactory : WebApplicationFactory<Program>
{
    // Зберігаємо набір мок-об'єктів для можливості перевірки у тестах
    public Mock<IProductRepository> ProductRepositoryMock { get; } = new();
    public Mock<IEmailService> EmailServiceMock { get; } = new();

    protected override void ConfigureWebHost(IWebHostBuilder builder)
    {
        builder.ConfigureTestServices(services =>
        {
            // Заміна всіх залежностей, що потребують ізоляції
            ReplaceService<IProductRepository>(services, ProductRepositoryMock.Object);
            ReplaceService<IEmailService>(services, EmailServiceMock.Object);
        });
    }

    private static void ReplaceService<TService>(IServiceCollection services, TService implementation)
        where TService : class
    {
        var descriptor = services.FirstOrDefault(d => d.ServiceType == typeof(TService));
        if (descriptor is not null) services.Remove(descriptor);
        services.AddScoped<TService>(_ => implementation);
    }
}
```

Використання кастомної фабрики:

```csharp [MyApp.Tests/Integration/ProductsApiTests.cs — з кастомною фабрикою]
public class ProductsApiTests(CustomWebApplicationFactory factory)
    : IClassFixture<CustomWebApplicationFactory>
{
    private readonly HttpClient _client = factory.CreateClient();

    [Fact]
    public async Task GetProduct_ExistingId_ReturnsOk()
    {
        // Налаштовуємо поведінку через Mock, що доступний у фабриці
        factory.ProductRepositoryMock
            .Setup(r => r.GetByIdAsync(42))
            .ReturnsAsync(new Product(42, "Monitor", 399m));

        var response = await _client.GetAsync("/products/42");

        Assert.Equal(HttpStatusCode.OK, response.StatusCode);
    }
}
```

## 7. POST-запити та десеріалізація JSON

Інтеграційні тести охоплюють і запити з тілом:

```csharp showLineNumbers
[Fact]
public async Task CreateProduct_ValidData_Returns201WithLocation()
{
    // Arrange
    factory.ProductRepositoryMock
        .Setup(r => r.SaveAsync(It.IsAny<Product>()))
        .ReturnsAsync(new Product(100, "New GPU", 699m));

    var newProduct = new { Name = "New GPU", Price = 699m };

    // Act — серіалізуємо об'єкт у JSON і відправляємо POST
    var response = await _client.PostAsJsonAsync("/products", newProduct);

    // Assert
    Assert.Equal(HttpStatusCode.Created, response.StatusCode);

    // Перевіряємо Location header
    Assert.NotNull(response.Headers.Location);
    Assert.Contains("/products/", response.Headers.Location.ToString());

    // Десеріалізуємо тіло відповіді
    var created = await response.Content.ReadFromJsonAsync<Product>();
    Assert.NotNull(created);
    Assert.Equal(100, created.Id);
}
```

## 8. Резюме

Інтеграційне тестування з `WebApplicationFactory` дає нам:

| Що тестуємо | Як |
|---|---|
| HTTP маршрутизацію та статуси | `GetAsync`, `PostAsJsonAsync` + `response.StatusCode` |
| Серіалізацію/десеріалізацію JSON | `ReadFromJsonAsync<T>()` |
| Middleware (авторизація, CORS) | Реальні HTTP-заголовки |
| Взаємодію між компонентами | Через заміну залежностей у `ConfigureTestServices` |

Ключові концепції: `IClassFixture<T>` для спільного сервера, `ConfigureTestServices` для заміни DI, кастомний `WebApplicationFactory` для перевикористання.
