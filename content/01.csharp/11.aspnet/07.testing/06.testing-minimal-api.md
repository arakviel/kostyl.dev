# Тестування Minimal API — авторизація, валідація та реальні сценарії

## 1. Від теорії до практики: Що ми будуємо

Ми пройшли довгий шлях: теорія тестування, xUnit, TDD, Moq, WebApplicationFactory. Настав час зібрати **всі знання разом** і протестувати реальний ASP.NET Minimal API — такий, як виглядають production-застосунки.

У цьому розділі ми будемо тестувати API для управління нотатками (Notes API) з:

- **Автентифікацією через JWT** — деякі ендпоінти захищені
- **Валідацією вхідних даних** — API відхиляє некоректні запити
- **CRUD-операціями** — повний набір ендпоінтів

Спочатку — сам API:

```csharp [MyApp/Program.cs] showLineNumbers
using Microsoft.AspNetCore.Authentication.JwtBearer;
using Microsoft.IdentityModel.Tokens;
using System.Text;

var builder = WebApplication.CreateBuilder(args);

builder.Services.AddAuthentication(JwtBearerDefaults.AuthenticationScheme)
    .AddJwtBearer(options =>
    {
        options.TokenValidationParameters = new TokenValidationParameters
        {
            ValidateIssuer = false,
            ValidateAudience = false,
            ValidateLifetime = false, // для тестів зручно вимкнути
            IssuerSigningKey = new SymmetricSecurityKey(
                Encoding.UTF8.GetBytes(builder.Configuration["Jwt:Key"] ?? "test-secret-key-32-chars-minimum!!"))
        };
    });

builder.Services.AddAuthorization();
builder.Services.AddScoped<INoteRepository, InMemoryNoteRepository>();

var app = builder.Build();
app.UseAuthentication();
app.UseAuthorization();

// Публічний ендпоінт
app.MapGet("/api/notes/public", () => Results.Ok(new[] { "Note 1", "Note 2" }));

// Захищені ендпоінти
var notes = app.MapGroup("/api/notes").RequireAuthorization();

notes.MapGet("/", async (INoteRepository repo) =>
    Results.Ok(await repo.GetAllAsync()));

notes.MapGet("/{id}", async (int id, INoteRepository repo) =>
{
    var note = await repo.GetByIdAsync(id);
    return note is null ? Results.NotFound() : Results.Ok(note);
});

notes.MapPost("/", async (CreateNoteDto dto, INoteRepository repo) =>
{
    if (string.IsNullOrWhiteSpace(dto.Title))
        return Results.ValidationProblem(new Dictionary<string, string[]>
        {
            ["Title"] = ["Заголовок не може бути порожнім."]
        });

    if (dto.Title.Length > 200)
        return Results.ValidationProblem(new Dictionary<string, string[]>
        {
            ["Title"] = ["Заголовок не може перевищувати 200 символів."]
        });

    var note = await repo.CreateAsync(dto);
    return Results.Created($"/api/notes/{note.Id}", note);
});

notes.MapDelete("/{id}", async (int id, INoteRepository repo) =>
{
    var deleted = await repo.DeleteAsync(id);
    return deleted ? Results.NoContent() : Results.NotFound();
});

app.Run();
public partial class Program { }
```

## 3. Тестування JWT-авторизації

Для тестів із JWT нам потрібно **генерувати тестові токени**. Виділимо це у допоміжний клас:

```csharp [MyApp.Tests/Infrastructure/TestJwtHelper.cs] showLineNumbers
using System.IdentityModel.Tokens.Jwt;
using System.Security.Claims;
using System.Text;
using Microsoft.IdentityModel.Tokens;

namespace MyApp.Tests.Infrastructure;

public static class TestJwtHelper
{
    private const string SecretKey = "test-secret-key-32-chars-minimum!!";

    public static string GenerateToken(int userId = 1, string role = "User")
    {
        var key = new SymmetricSecurityKey(Encoding.UTF8.GetBytes(SecretKey));
        var credentials = new SigningCredentials(key, SecurityAlgorithms.HmacSha256);

        var claims = new[]
        {
            new Claim(ClaimTypes.NameIdentifier, userId.ToString()),
            new Claim(ClaimTypes.Role, role),
        };

        var token = new JwtSecurityToken(
            claims: claims,
            expires: DateTime.UtcNow.AddHours(1),
            signingCredentials: credentials);

        return new JwtSecurityTokenHandler().WriteToken(token);
    }

    // Зручний метод для додавання заголовку до HttpClient
    public static HttpClient WithBearerToken(this HttpClient client, string? token = null)
    {
        var jwt = token ?? GenerateToken();
        client.DefaultRequestHeaders.Authorization =
            new System.Net.Http.Headers.AuthenticationHeaderValue("Bearer", jwt);
        return client;
    }
}
```

## 4. Тестування валідації вхідних даних

```csharp [MyApp.Tests/Integration/NotesValidationTests.cs] showLineNumbers
using System.Net;
using System.Net.Http.Json;
using MyApp.Tests.Infrastructure;

namespace MyApp.Tests.Integration;

public class NotesValidationTests(WebApplicationFactory<Program> factory)
    : IClassFixture<WebApplicationFactory<Program>>
{
    private HttpClient AuthorizedClient =>
        factory.CreateClient().WithBearerToken();

    [Fact]
    public async Task CreateNote_EmptyTitle_Returns422ValidationProblem()
    {
        // Arrange
        var invalidDto = new { Title = "", Content = "Якийсь текст" };

        // Act
        var response = await AuthorizedClient.PostAsJsonAsync("/api/notes", invalidDto);

        // Assert
        Assert.Equal(HttpStatusCode.UnprocessableEntity, response.StatusCode);

        // Перевіряємо тіло помилки валідації (ValidationProblem)
        var body = await response.Content.ReadFromJsonAsync<ValidationProblemDetails>();
        Assert.NotNull(body);
        Assert.True(body.Errors.ContainsKey("Title"));
        Assert.Contains("порожнім", body.Errors["Title"][0]);
    }

    [Fact]
    public async Task CreateNote_TitleTooLong_Returns422WithTitleError()
    {
        // Arrange — заголовок у 201 символів (ліміт 200)
        var invalidDto = new { Title = new string('A', 201), Content = "" };

        // Act
        var response = await AuthorizedClient.PostAsJsonAsync("/api/notes", invalidDto);

        // Assert
        Assert.Equal(HttpStatusCode.UnprocessableEntity, response.StatusCode);
        var body = await response.Content.ReadFromJsonAsync<ValidationProblemDetails>();
        Assert.True(body!.Errors.ContainsKey("Title"));
    }

    [Theory]
    [InlineData("Нормальний заголовок")]
    [InlineData("A")]
    [InlineData("Заголовок з символами! @#$%")]
    public async Task CreateNote_ValidTitle_Returns201(string validTitle)
    {
        // Arrange
        var validDto = new { Title = validTitle, Content = "Вміст нотатки" };

        // Act
        var response = await AuthorizedClient.PostAsJsonAsync("/api/notes", validDto);

        // Assert
        Assert.Equal(HttpStatusCode.Created, response.StatusCode);
        Assert.NotNull(response.Headers.Location);
    }
}
```

## 5. Повний CRUD-цикл: Сценарний тест

Деколи важливо перевірити **повний користувацький сценарій** — від створення до видалення. Такі тести одночасно демонструють і роботу системи, і слугують живою документацією:

```csharp [MyApp.Tests/Integration/NotesFullCycleTests.cs] showLineNumbers
using System.Net;
using System.Net.Http.Json;
using MyApp.Tests.Infrastructure;

namespace MyApp.Tests.Integration;

public class NotesFullCycleTests(WebApplicationFactory<Program> factory)
    : IClassFixture<WebApplicationFactory<Program>>
{
    private HttpClient AuthorizedClient =>
        factory.CreateClient().WithBearerToken();

    [Fact]
    public async Task Notes_FullCrudCycle_WorksCorrectly()
    {
        var client = AuthorizedClient;

        // === КРОК 1: Читаємо порожній список ===
        var listResponse = await client.GetAsync("/api/notes");
        Assert.Equal(HttpStatusCode.OK, listResponse.StatusCode);
        var initialNotes = await listResponse.Content.ReadFromJsonAsync<Note[]>();
        var initialCount = initialNotes?.Length ?? 0;

        // === КРОК 2: Створюємо нотатку ===
        var createDto = new { Title = "Тестова нотатка", Content = "Вміст тесту" };
        var createResponse = await client.PostAsJsonAsync("/api/notes", createDto);
        Assert.Equal(HttpStatusCode.Created, createResponse.StatusCode);

        // Отримуємо створену нотатку
        var createdNote = await createResponse.Content.ReadFromJsonAsync<Note>();
        Assert.NotNull(createdNote);
        Assert.Equal("Тестова нотатка", createdNote.Title);
        var noteId = createdNote.Id;

        // === КРОК 3: Читаємо список — має бути на 1 більше ===
        var listAfterCreate = await client.GetAsync("/api/notes");
        var notesAfterCreate = await listAfterCreate.Content.ReadFromJsonAsync<Note[]>();
        Assert.Equal(initialCount + 1, notesAfterCreate?.Length);

        // === КРОК 4: Читаємо конкретну нотатку ===
        var getResponse = await client.GetAsync($"/api/notes/{noteId}");
        Assert.Equal(HttpStatusCode.OK, getResponse.StatusCode);

        // === КРОК 5: Видаляємо нотатку ===
        var deleteResponse = await client.DeleteAsync($"/api/notes/{noteId}");
        Assert.Equal(HttpStatusCode.NoContent, deleteResponse.StatusCode);

        // === КРОК 6: Нотатка більше не існує ===
        var afterDeleteResponse = await client.GetAsync($"/api/notes/{noteId}");
        Assert.Equal(HttpStatusCode.NotFound, afterDeleteResponse.StatusCode);
    }
}
```

## 6. Ізоляція між тестами: `IAsyncLifetime`

Коли тести використовують in-memory сховище, вони **можуть впливати одне на одного** (стан зберігається між тестами). Для очищення стану використовується `IAsyncLifetime`:

```csharp [MyApp.Tests/Integration/IsolatedNotesTests.cs] showLineNumbers
namespace MyApp.Tests.Integration;

public class IsolatedNotesTests
    : IClassFixture<WebApplicationFactory<Program>>, IAsyncLifetime
{
    private readonly WebApplicationFactory<Program> _factory;
    private HttpClient _client = null!;

    public IsolatedNotesTests(WebApplicationFactory<Program> factory)
    {
        _factory = factory;
    }

    // Виконується перед кожним тестом (аналог [SetUp] у NUnit)
    public async Task InitializeAsync()
    {
        _client = _factory.CreateClient().WithBearerToken();
        // Тут можна очистити БД або підготувати тестові дані
        await _client.DeleteAsync("/api/notes/all"); // гіпотетичний ендпоінт для тестів
    }

    // Виконується після кожного тесту (аналог [TearDown])
    public Task DisposeAsync() => Task.CompletedTask;

    [Fact]
    public async Task Test_AlwaysStartsWithCleanState()
    {
        var response = await _client.GetAsync("/api/notes");
        var notes = await response.Content.ReadFromJsonAsync<Note[]>();
        Assert.Empty(notes ?? []);
    }
}
```

## 8. Резюме розділу «Тестування»

Ми пройшли повний шлях від нічого до production-рівневого тестування:

::steps

### Крок 1 — Основи (розділ 1)

Зрозуміли **навіщо** тести — піраміда тестування, AAA-патерн, термінологія (SUT, Mock, Stub, Fake, Regression).

### Крок 2 — xUnit (розділ 2)

Навчилися писати перші тести з `[Fact]`, `[Theory]`, `Assert.*`, налаштовувати тестовий проєкт та правильно іменувати тести.

### Крок 3 — TDD (розділ 3)

Засвоїли методологію **Red → Green → Refactor**: написання тесту до реалізації змінює мислення розробника та покращує дизайн коду.

### Крок 4 — Moq (розділ 4)

Навчилися ізолювати залежності через мок-об'єкти: `Setup` для поведінки, `Verify` для перевірки взаємодій, `It.*` для гнучких матчерів.

### Крок 5 — WebApplicationFactory (розділ 5)

Опанували інтеграційне тестування: реальний HTTP-клієнт проти тестового in-memory сервера, заміна залежностей через `ConfigureTestServices`.

### Крок 6 — Реальні сценарії (цей розділ)

Поєднали всі знання: тестування JWT-авторизації, валідації, CRUD-сценаріїв та ізоляції між тестами.
