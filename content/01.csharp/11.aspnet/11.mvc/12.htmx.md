# HTMX: HTML-driven інтерактивність без JavaScript фреймворків

Це 2012 рік. Щоразу як ви клікаєте кнопку на сторінці, браузер перезавантажує її повністю. Сервер генерує весь HTML заново і надсилає його клієнту. Просто і передбачувано.

Потім прийшли React, Vue, Angular: замість HTML сервер повертає JSON, а клієнтський JavaScript будує DOM. За це ми платимо сотнями кілобайтів JavaScript, складністю стану на клієнті і принципово розділеним фронтендом від бекенду.

**HTMX** пропонує третій шлях. Його філософія — «**HTML over the wire**»: сервер повертає не JSON, а готовий HTML-фрагмент, який браузер вставляє у DOM без перезавантаження сторінки. AJAX є, але JavaScript писати не треба — усе управляється HTML-атрибутами.

---

## Підключення

HTMX — це єдиний JavaScript файл розміром ~14kb (gzip). Жодних залежностей, жодного npm:

```html [Views/Shared/_Layout.cshtml]
@* Або через CDN *@
<script src="https://unpkg.com/htmx.org@2.0.3"
        integrity="sha384-..." crossorigin="anonymous"></script>

@* Або локально (рекомендовано для production) *@
<script src="~/lib/htmx/htmx.min.js"></script>
```

Після підключення HTMX активний для всього документа. Ви починаєте додавати `hx-*` атрибути до існуючих HTML-елементів — і вони стають інтерактивними.

## Core атрибути: розбір кожного

### `hx-get`, `hx-post`, `hx-put`, `hx-patch`, `hx-delete` — HTTP-метод і URL

Найважливіші атрибути — визначають який HTTP-запит відправити і куди:

```html
@* GET-запит на /articles при кліку *@
<button hx-get="/articles">Завантажити статті</button>

@* POST-запит на /articles/create при кліку *@
<button hx-post="/articles/create">Створити</button>

@* DELETE при кліку на кнопку видалення *@
<button hx-delete="/articles/42">Видалити</button>
```

За замовчуванням HTMX вставляє HTML-відповідь **всередину елемента** що ініціював запит (innerHTML). Тобто після кліку на `<button hx-get="/articles">` — вміст кнопки замінюється відповіддю сервера. Це рідко те що потрібно — і тут на допомогу приходить `hx-target`.

---

### `hx-target` — куди вставляти відповідь

Вказує CSS-селектор елемента куди вставити HTML-відповідь:

```html
@* Відповідь вставляється у #results, а не у кнопку *@
<button hx-get="/articles" hx-target="#results">
    Завантажити статті
</button>

<div id="results">
    @* Сюди прийде HTML від сервера *@
</div>
```

**Спеціальні значення `hx-target`:**

| Значення | Що означає |
|---|---|
| `#element-id` | Елемент за CSS-ідентифікатором |
| `.class-name` | Перший елемент за класом |
| `closest tr` | Найближчий батьківський `<tr>` |
| `next div` | Наступний сусідній `<div>` |
| `previous p` | Попередній сусідній `<p>` |
| `this` | Сам елемент що ініціював запит (за замовчуванням) |

### `hx-swap` — як саме вставляти

`hx-swap` визначає **де саме** відносно `hx-target` вставляється відповідь:

```html
<button hx-get="/articles/new"
        hx-target="#list"
        hx-swap="beforeend">        @*← додати наприкінці списку *@
    Завантажити ще
</button>
```

### `hx-trigger` — коли надсилати запит

За замовчуванням HTMX реагує на природні події: `click` для кнопок, `submit` для форм, `change` для `<select>`. `hx-trigger` дозволяє налаштувати будь-яку подію:

```html
@* Пошук при вводі ("input") з затримкою 300мс *@
<input type="search"
       hx-get="/search"
       hx-target="#results"
       hx-trigger="input changed delay:300ms"
       name="q"
       placeholder="Пошук...">

@* Завантажити при першій появі у вьюпорті *@
<div hx-get="/lazy-section" hx-trigger="revealed">
    Завантаження...
</div>

@* Кожні 5 секунд автоматично оновлювати *@
<div hx-get="/notifications/count"
     hx-trigger="every 5s"
     hx-target="#notification-badge">
</div>

@* Після кастомної події від іншого елемента *@
<div hx-get="/cart/summary"
     hx-trigger="cartUpdated from:body">
</div>
```

**Модифікатори `hx-trigger`:**

| Модифікатор | Значення |
|---|---|
| `delay:300ms` | Затримка перед відправкою |
| `throttle:1s` | Не більше одного запиту за 1 секунду |
| `changed` | Лише якщо значення змінилося |
| `once` | Запит лише один раз |
| `revealed` | При появі у вьюпорті |
| `from:#id` | Подія від іншого елемента за CSS-селектором |
| `every 5s` | Polling — повторювати з інтервалом |

### `hx-include` — додаткові поля до запиту

`hx-include` серіалізує значення вказаних елементів і додає їх до запиту:

```html
@* Включити значення фільтру при натисні кнопки *@
<select id="category-filter" name="category">
    <option value="">Всі</option>
    <option value="tech">Технології</option>
    <option value="design">Дизайн</option>
</select>

<button hx-get="/articles"
        hx-target="#articles-list"
        hx-include="#category-filter">  @* ← додає category=tech до запиту *@
    Фільтрувати
</button>
```

---

### `hx-indicator` — індикатор завантаження

Поки запит виконується — HTMX додає CSS-клас `htmx-request` до елемента-ініціатора. `hx-indicator` дозволяє показати/приховати конкретний спіннер:

```html
<button hx-get="/slow-operation"
        hx-indicator="#spinner">
    Виконати
    <span id="spinner" class="htmx-indicator">
        <i class="bi bi-arrow-repeat spinner-icon"></i>
    </span>
</button>
```

```css
.htmx-indicator { display: none; }
.htmx-request .htmx-indicator { display: inline-block; }
.htmx-request.htmx-indicator { display: inline-block; }
```

За замовчуванням, поки запит виконується, на елементі є клас `htmx-request` — тому кнопку можна стилізувати:

```css
.htmx-request { opacity: 0.7; pointer-events: none; cursor: wait; }
```

### `hx-push-url` — URL у адресному рядку

При AJAX-запитах URL у браузері не змінюється. `hx-push-url` виправляє це — браузер додає запис до history:

```html
@* При кліку на статтю — URL змінюється на /articles/42 *@
<a hx-get="/articles/42"
   hx-target="#content"
   hx-push-url="/articles/42">
    Відкрити статтю
</a>
```

Тепер кнопка «Назад» браузера повертає попередній стан — HTMX перехопить `popstate` і відновить вміст.

---

### `hx-boost` — прискорення звичайних посилань

`hx-boost="true"` перетворює звичайні `<a href>` та `<form>` на HTMX-запити автоматично:

```html
@* Всі посилання всередині nav стають HTMX-запитами *@
<nav hx-boost="true">
    <a href="/articles">Статті</a>      @* → hx-get="/articles" hx-target="body" *@
    <a href="/about">Про нас</a>
</nav>
```

Замість перезавантаження сторінки HTMX робить AJAX і замінює `<body>`. Ефект — SPA-навігація без жодного JavaScript.

## Out-of-band Swaps: оновлення кількох елементів з одного запиту

Стандартно HTMX вставляє відповідь лише у `hx-target`. Але що якщо потрібно **оновити кілька елементів** одночасно — наприклад, лічильник кошика в навбарі і список товарів на сторінці?

Для цього є **Out-of-Band (OOB) Swaps**: сервер повертає HTML з атрибутом `hx-swap-oob`:

```html
@* Відповідь сервера може містити кілька фрагментів *@

@* Основний фрагмент — вставляється у hx-target *@
<div id="cart-items">
    <p>Додано: Навушники Sony WH-1000XM5</p>
</div>

@* OOB-фрагмент — вставляється у #cart-count незалежно від hx-target *@
<span id="cart-count" hx-swap-oob="true">3</span>

@* Ще один OOB-фрагмент *@
<div id="notification-toast" hx-swap-oob="innerHTML">
    Товар додано до кошика!
</div>
```

Сервер повертає весь цей HTML в одній відповіді. HTMX розбирає її, вставляє основний фрагмент у `hx-target`, а OOB-елементи знаходить за їх `id` та оновлює їх у DOM незалежно.

## Server-Sent Events (SSE) з HTMX

HTMX має вбудовану підтримку SSE (одностороннє з'єднання сервер→клієнт):

```html
@* Підписатися на SSE потік з /notifications/stream *@
<div hx-ext="sse" sse-connect="/notifications/stream">
    <div sse-swap="new-notification" hx-target="#notification-list" hx-swap="beforeend">
        @* Нові сповіщення будуть дописуватися сюди *@
    </div>
</div>
```

Потрібно підключити SSE extension:

```html
<script src="https://unpkg.com/htmx-ext-sse@2.2.2/sse.js"></script>
```

## `hx-boost` vs React vs Alpine.js: порівняння

| | Vanilla HTMX | `hx-boost` | Alpine.js | React |
|---|---|---|---|---|
| Розмір | 14kb | 14kb | 15kb | 45kb+ |
| JS-код що пишете ви | Мінімум | Нуль | Небагато | Багато |
| Стан клієнта | Немає | Немає | Локальний | Складний |
| Server-side rendering | Завжди | Завжди | Гібрид | Опціонально |
| SEO | ✅ | ✅ | ✅ | Вимагає SSR |
| Складні SPA | ❌ | ❌ | Частково | ✅ |
| Реальний час (WebSocket) | Через ext | Через ext | Вручну | Вручну |
| Крива навчання | Мінімальна | Мінімальна | Середня | Висока |

HTMX — не конкурент React. Це інструмент для **сервер-рендерингу з інтерактивністю**: форми, таблиці, пошук, фільтрація, модальні вікна. Для складних клієнтських додатків (редактори, дашборди з реальним часом) React/Vue залишаються кращим вибором.

## Демо: живий список задач

Побудуємо повністю функціональний To-Do список без перезавантаження сторінки — лише HTMX.

### Controller

```csharp [Controllers/TodoController.cs]
using Microsoft.AspNetCore.Mvc;

namespace HtmxDemo.Controllers;

public class TodoController : Controller
{
    // In-memory storage для демо
    private static readonly List<TodoItem> _todos =
    [
        new(1, "Вивчити HTMX", false),
        new(2, "Написати todo-апку", false),
        new(3, "Зробити каву", true),
    ];
    private static int _nextId = 4;

    // GET /todo → повна сторінка
    public IActionResult Index() => View(_todos.AsReadOnly());

    // GET /todo/list → лише список (для HTMX)
    public IActionResult List() =>
        PartialView("_TodoList", _todos.AsReadOnly());

    // POST /todo/create?text=...
    [HttpPost]
    public IActionResult Create(string text)
    {
        if (string.IsNullOrWhiteSpace(text))
            return BadRequest();

        var item = new TodoItem(_nextId++, text.Trim(), false);
        _todos.Add(item);

        // Повертаємо лише новий рядок — HTMX вставить його до списку
        return PartialView("_TodoItem", item);
    }

    // PUT /todo/{id}/toggle
    [HttpPost, Route("todo/{id}/toggle")]
    public IActionResult Toggle(int id)
    {
        var item = _todos.FirstOrDefault(t => t.Id == id);
        if (item is null) return NotFound();

        var index = _todos.IndexOf(item);
        _todos[index] = item with { IsCompleted = !item.IsCompleted };

        return PartialView("_TodoItem", _todos[index]);
    }

    // DELETE /todo/{id}/delete
    [HttpDelete, Route("todo/{id}/delete")]
    public IActionResult Delete(int id)
    {
        _todos.RemoveAll(t => t.Id == id);
        // hx-swap="outerHTML" + порожня відповідь = видалення елемента
        return Ok();
    }
}

public record TodoItem(int Id, string Text, bool IsCompleted);
```

### Views

```html [Views/Todo/Index.cshtml]

        </button>
    </form>

    @* Список задач *@

    </ul>

```

```html [Views/Todo/_TodoItem.cshtml]

@* id="todo-{id}" — щоб Toggle міг замінити конкретний рядок через outerHTML *@
<li id="todo-@Model.Id"
    class="list-group-item d-flex align-items-center gap-3">

    @* Чекбокс: PUT /todo/{id}/toggle при зміні *@

           hx-target="#todo-@Model.Id"     @* замінити весь <li> *@
           hx-swap="outerHTML">

        @Model.Text
    </span>

    @* Кнопка видалення: DELETE /todo/{id}/delete *@
    <button class="btn btn-sm btn-outline-danger ms-auto"
            hx-delete="/todo/@Model.Id/delete"
            hx-target="#todo-@Model.Id"     @* видалити цей <li> *@
            hx-swap="outerHTML"
            hx-confirm="Видалити задачу?">  @* нативний confirm-діалог *@
        <i class="bi bi-trash"></i>
    </button>
</li>
```

Весь інтерактивний список задач — **без єдиного рядка JavaScript**, написаного вами. HTMX обробляє всі запити, оновлює DOM, очищує форму.

## Практичні завдання

### Рівень 1 — Базовий

**Завдання 1.1.** Додайте до Todo-демо **фільтрацію**: три кнопки «Всі», «Активні», «Виконані». При кліку — GET `/todo/list?filter=active` (або `all`, `completed`). Сервер повертає відфільтрований Partial View `_TodoList`. Активна кнопка підсвічується — використайте `hx-swap-oob` для оновлення кнопок.

**Завдання 1.2.** Реалізуйте ефект **«Toast notification»** після кожної дії (додати, видалити, завершити): Controller повертає OOB-фрагмент `<div id="toast" hx-swap-oob="true">...повідомлення...</div>`. Toast автоматично зникає через 3 секунди через CSS-анімацію та `setTimeout` у одному рядку JavaScript.

### Рівень 2 — Логіка

**Завдання 2.1.** Реалізуйте **live-search** для списку статей: `<input hx-get="/articles/search" hx-trigger="input changed delay:300ms" hx-target="#articles">`. Controller повертає Partial View зі відфільтрованими статтями. Додайте `hx-indicator` зі спіннером та `hx-push-url="/articles?q={query}"` щоб URL оновлювався і пошук можна було закладинкувати.

**Завдання 2.2.** Побудуйте **lazy-завантаження** сторінок таблиці (`hx-trigger="revealed"`): перша сторінка рендериться на сервері, в кінці списку є порожній `<div hx-get="/articles?page=2" hx-trigger="revealed" hx-swap="outerHTML">`. При скролі донизу HTMX автоматично підвантажує наступну сторінку, замінюючи div на нові рядки + новий sentinel-div для наступної сторінки.

### Рівень 3 — Архітектура

**Завдання 3.1.** Реалізуйте **real-time лічильник online-користувачів** через SSE:
- `GET /realtime/users-online` — SSE endpoint що кожні 5 секунд надсилає `event: users-count\ndata: <span>N online</span>\n\n`
- View підключається через `hx-ext="sse" sse-connect="/realtime/users-online"` + `sse-swap="users-count" hx-swap="innerHTML"`
- На сервері — `IHostedService` що зберігає кількість активних з'єднань (Interlocked.Increment/Decrement при connect/disconnect)

## Резюме

- **HTMX** — JavaScript-бібліотека (14kb) для побудови інтерактивного UI через HTML-атрибути. Сервер повертає **HTML**, а не JSON
- **`hx-get/post/put/patch/delete`** — HTTP-метод та URL запиту
- **`hx-target`** — CSS-селектор елемента куди вставляти відповідь
- **`hx-swap`** — як вставляти: `innerHTML`, `outerHTML`, `beforeend`, `afterend`, `delete`, `none`
- **`hx-trigger`** — подія: `click`, `input delay:300ms`, `revealed`, `every 5s`, `from:#id`
- **`hx-include`** — включити значення додаткових полів до запиту
- **`hx-indicator`** — елемент спіннера активується класом `htmx-request`
- **`hx-push-url`** — оновлює URL у браузері без перезавантаження
- **`hx-boost`** — перетворює всі `<a>` та `<form>` у HTMX-запити автоматично
- **OOB Swaps** (`hx-swap-oob`) — оновлення кількох елементів DOM з однієї відповіді
- **SSE** — `hx-ext="sse"` + `sse-connect` + `sse-swap` для real-time оновлень

У наступній статті — **HTMX у MVC**: `JsonResult` vs `PartialViewResult`, AntiForgery з HTMX, `HX-Redirect`, `HX-Trigger` заголовки та практичні патерни: live-search, lazy loading, inline edit.
