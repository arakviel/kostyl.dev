# HTMX у MVC: інтерактивний UI без SPA

Попередня стаття описала HTMX як технологію — атрибути, тригери, swap-режими. Тепер розберемо **інтеграцію з ASP.NET Core MVC**: як правильно організовувати Controller-методи, що повертають HTML-фрагменти, як захистити форми від CSRF, як передавати інструкції браузеру через HTTP-заголовки.

HTMX у MVC — це патерн де **Controller повертає не повну сторінку, а лише фрагмент HTML** (Partial View), який HTMX вставляє у потрібне місце DOM. Логіка залишається на сервері, JavaScript — мінімальний.

## Виявлення HTMX-запитів: заголовок `HX-Request`

Перше що потрібно зрозуміти: HTMX автоматично додає до кожного свого запиту заголовок `HX-Request: true`. Це дозволяє Controller визначити — звичайний navigation запит чи HTMX-запит — і повернути відповідно повну сторінку або лише фрагмент:

```csharp [Controllers/ArticleController.cs]
using Microsoft.AspNetCore.Mvc;

namespace BlogApp.Controllers;

public class ArticleController : Controller
{
    private readonly IArticleService _articles;

    public ArticleController(IArticleService articles)
    {
        _articles = articles;
    }

    public async Task<IActionResult> Index(string? q)
    {
        var articles = await _articles.SearchAsync(q);

        // HTMX-запит → лише список без _Layout
        if (Request.Headers.ContainsKey("HX-Request"))
            return PartialView("_ArticleList", articles);

        // Звичайний браузерний запит → повна сторінка з _Layout
        return View(articles);
    }
}
```

Для зручності винесемо перевірку у extension-метод:

```csharp [Extensions/HttpRequestExtensions.cs]
namespace BlogApp.Extensions;

public static class HttpRequestExtensions
{
    // true якщо запит надійшов від HTMX
    public static bool IsHtmx(this HttpRequest request)
        => request.Headers.ContainsKey("HX-Request");

    // URL поточної сторінки в браузері (звідки надійшов HTMX-запит)
    public static string? HtmxCurrentUrl(this HttpRequest request)
        => request.Headers.TryGetValue("HX-Current-URL", out var val) ? val.ToString() : null;

    // Target-елемент (значення hx-target на клієнті)
    public static string? HtmxTarget(this HttpRequest request)
        => request.Headers.TryGetValue("HX-Target", out var val) ? val.ToString() : null;
}
```

```csharp [Controllers/ArticleController.cs — з extension]
using BlogApp.Extensions;

public async Task<IActionResult> Index(string? q)
{
    var articles = await _articles.SearchAsync(q);

    return Request.IsHtmx()
        ? PartialView("_ArticleList", articles)
        : View(articles);
}
```

## AntiForgery Token з HTMX

ASP.NET Core захищає POST/PUT/DELETE форми через AntiForgery token. Стандартна форма включає `@Html.AntiForgeryToken()` автоматично через `<form asp-action="...">`. З HTMX є два підходи.

**Підхід 1: Форmic HTMX-форми** — за замовчуванням HTMX серіалізує весь вміст форми включно з прихованим CSRF-полем:

```html
@* HTMX + форма: antiforgery token передається автоматично бо вся форма серіалізується *@
<form hx-post="/article/create" hx-target="#result">
    @Html.AntiForgeryToken()   @* ← прихований input __RequestVerificationToken *@
    <input name="Title" type="text">
    <button type="submit">Створити</button>
</form>
```

**Підхід 2: hx-vals або Meta-тег** — для запитів що не є формами (кнопки, посилання), передаємо токен через `hx-vals` або глобальний `htmx:configRequest`:

```html [Views/Shared/_Layout.cshtml]
@* Глобальний meta-тег з CSRF-токеном *@
@inject Microsoft.AspNetCore.Antiforgery.IAntiforgery Antiforgery
@{
    var token = Antiforgery.GetAndStoreTokens(Context).RequestToken;
}
<meta name="csrf-token" content="@token">

<script>
    // HTMX: додавати CSRF-заголовок до кожного non-GET запиту
    document.addEventListener("htmx:configRequest", function(e) {
        if (e.detail.verb !== "get") {
            e.detail.headers["RequestVerificationToken"] =
                document.querySelector("meta[name='csrf-token']").content;
        }
    });
</script>
```

```csharp [Program.cs — налаштування AntiForgery для заголовків]
builder.Services.AddAntiforgery(options =>
{
    // ASP.NET Core прийматиме токен з цього заголовка
    options.HeaderName = "RequestVerificationToken";
});
```

Тепер `hx-delete="/article/42"` на звичайній кнопці автоматично отримає CSRF-захист.

## HX-Response Headers: керування браузером з сервера

HTMX читає певні HTTP-заголовки у відповіді та виконує відповідні дії. Це «зворотний» канал — сервер керує поведінкою HTMX без JavaScript.

### `HX-Redirect` — перенаправлення

Замість `302 Redirect` що перезавантажує сторінку, `HX-Redirect` говорить HTMX зробити навігацію на нову URL:

```csharp [Controllers/ArticleController.cs]
[HttpPost]
[ValidateAntiForgeryToken]
public async Task<IActionResult> Create(CreateArticleDto dto)
{
    if (!ModelState.IsValid)
        return PartialView("_CreateForm", dto); // повертаємо форму з помилками

    var article = await _articles.CreateAsync(dto);

    // HX-Redirect: HTMX зробить navigation без перезавантаження сторінки
    Response.Headers["HX-Redirect"] = $"/articles/{article.Id}";
    return Ok(); // відповідь 200, але тіло ігнорується — HTMX перенаправляє
}
```

### `HX-Trigger` — виклик клієнтських подій

`HX-Trigger` у заголовку відповіді ініціює кастомну подію на клієнті. Інші HTMX-елементи можуть слухати цю подію:

```csharp [Controllers/CartController.cs]
[HttpPost]
public async Task<IActionResult> AddToCart(int productId, int quantity)
{
    await _cart.AddAsync(productId, quantity);
    var count = await _cart.GetCountAsync();

    // Ініціювати подію "cartUpdated" з даними
    Response.Headers["HX-Trigger"] = $"{{\"cartUpdated\": {{\"count\": {count}}}}}";

    return PartialView("_AddedToCartConfirmation", productId);
}
```

```html
@* Елемент слухає подію "cartUpdated" *@
<span id="cart-badge"
      hx-get="/cart/count"
      hx-trigger="cartUpdated from:body"
      hx-target="this"
      hx-swap="innerHTML">
    0
</span>
```

### `HX-Retarget` та `HX-Reswap` — зміна цілі з сервера

Іноді сервер хоче змінити куди і як вставити відповідь — незалежно від того що вказано в `hx-target`/`hx-swap` на клієнті:

```csharp [Controllers/ArticleController.cs]
public async Task<IActionResult> Create(CreateArticleDto dto)
{
    if (!ModelState.IsValid)
    {
        // Помилка: показати форму у іншому місці від очікуваного
        Response.Headers["HX-Retarget"] = "#error-container";
        Response.Headers["HX-Reswap"] = "innerHTML";
        Response.StatusCode = 422; // Unprocessable Content — HTMX обробляє error responses
        return PartialView("_ValidationErrors", ModelState);
    }
    // ...
}
```

### Патерн 1: Live-Search

Живий пошук що оновлюється при введенні тексту з debounce 300ms.

```html [Views/Article/Index.cshtml]
<div class="mb-3">
    <input type="search"
           name="q"
           hx-get="/article/search"
           hx-trigger="input changed delay:300ms, search"
           hx-target="#articles-container"
           hx-indicator="#search-spinner"
           placeholder="Пошук статей..."
           class="form-control form-control-lg"
           autocomplete="off">
    <span id="search-spinner" class="htmx-indicator">
        <div class="spinner-border spinner-border-sm text-secondary mt-1" role="status">
            <span class="visually-hidden">Пошук...</span>
        </div>
    </span>
</div>

<div id="articles-container">
    <partial name="_ArticleList" model="Model"/>
</div>
```

```csharp [Controllers/ArticleController.cs]
// GET /article/search?q=htmx
public async Task<IActionResult> Search(string? q)
{
    var articles = string.IsNullOrWhiteSpace(q)
        ? await _articles.GetAllAsync()
        : await _articles.SearchAsync(q);

    // Завжди Partial — цей endpoint викликається лише через HTMX
    return PartialView("_ArticleList", articles);
}
```

```html [Views/Article/_ArticleList.cshtml]
@model IReadOnlyList<Article>

@if (!Model.Any())
{
    <p class="text-muted text-center py-4">Нічого не знайдено</p>
}
else
{
    @foreach (var article in Model)
    {
        <div class="card mb-3">
            <div class="card-body">
                <h5 class="card-title">@article.Title</h5>
                <p class="card-text text-muted">@article.Author · @article.PublishedAt.ToString("dd.MM.yyyy")</p>
            </div>
        </div>
    }
}
```

### Патерн 3: Inline Edit

Клік на клітинку таблиці → поле редагування → збереження без перезавантаження.

```html [Views/Product/_ProductRow.cshtml]
@model Product

<tr id="product-row-@Model.Id">
    <td>
        @* Клік на назву: замінити рядок на форму редагування *@
        <span class="editable"
              hx-get="/product/@Model.Id/edit"
              hx-target="#product-row-@Model.Id"
              hx-swap="outerHTML"
              title="Клікніть для редагування"
              style="cursor: pointer; border-bottom: 1px dashed #ccc;">
            @Model.Name
        </span>
    </td>
    <td>@Model.Price.Formatted</td>
    <td>@Model.Category</td>
</tr>
```

```html [Views/Product/_EditProductRow.cshtml]
@model Product

<tr id="product-row-@Model.Id">
    <td>
        @* Форма з inline редагуванням *@
        <form hx-patch="/product/@Model.Id"
              hx-target="#product-row-@Model.Id"
              hx-swap="outerHTML"
              class="d-flex gap-1">
            @Html.AntiForgeryToken()
            <input name="Name" value="@Model.Name" class="form-control form-control-sm"
                   autofocus required>
            <button type="submit" class="btn btn-sm btn-success">✓</button>
            @* Скасувати: повернути оригінальний рядок *@
            <button type="button" class="btn btn-sm btn-outline-secondary"
                    hx-get="/product/@Model.Id/row"
                    hx-target="#product-row-@Model.Id"
                    hx-swap="outerHTML">✕</button>
        </form>
    </td>
    <td>@Model.Price.Formatted</td>
    <td>@Model.Category</td>
</tr>
```

```csharp [Controllers/ProductController.cs]
// GET /product/{id}/edit → форма редагування (замінює рядок)
public async Task<IActionResult> Edit(int id)
{
    var product = await _products.GetByIdAsync(id);
    if (product is null) return NotFound();
    return PartialView("_EditProductRow", product);
}

// GET /product/{id}/row → оригінальний рядок (для скасування)
public async Task<IActionResult> Row(int id)
{
    var product = await _products.GetByIdAsync(id);
    if (product is null) return NotFound();
    return PartialView("_ProductRow", product);
}

// PATCH /product/{id} → зберегти зміни і повернути оновлений рядок
[HttpPatch]
[ValidateAntiForgeryToken]
public async Task<IActionResult> Update(int id, string name)
{
    var product = await _products.UpdateNameAsync(id, name);
    if (product is null) return NotFound();

    // HX-Trigger: повідомити інші елементи що продукт оновлено
    Response.Headers["HX-Trigger"] = "productUpdated";

    return PartialView("_ProductRow", product);
}
```

## Обробка помилок з HTMX

HTMX за замовчуванням обробляє лише відповіді з кодом `2xx`. Для `4xx`/`5xx` потрібно явно налаштувати:

```javascript
// Глобальний обробник помилок HTMX
document.addEventListener("htmx:responseError", function(e) {
    const status = e.detail.xhr.status;
    const target = e.detail.target;

    if (status === 422) {
        // Validation error — HTMX сам вставить відповідь, якщо ми повернули HTML
        // Нічого додаткового не потрібно якщо HX-Reswap встановлений на сервері
    } else if (status === 403) {
        alert("Доступ заборонено. Авторизуйтесь.");
    } else if (status >= 500) {
        // Показати toast
        document.getElementById("error-toast").textContent =
            "Помилка сервера. Спробуйте пізніше.";
    }
});
```

На сервері для HTMX-помилок краще повертати `422 Unprocessable Content` з HTML-фрагментом замість `400 Bad Request`:

```csharp
if (!ModelState.IsValid)
{
    Response.StatusCode = 422; // HTMX оброблятиме це як "error" але вставить тіло відповіді
    return PartialView("_FormWithErrors", dto);
}
```

## Практичні завдання

### Рівень 1 — Базовий

**Завдання 1.1.** Реалізуйте HTMX-форму коментаря до статті:
- `POST /article/{id}/comment` + `[ValidateAntiForgeryToken]`
- При успіху: повертає `_CommentItem` Partial View, HTMX вставляє через `hx-swap="beforeend"` до `#comments-list`
- При помилці (порожній текст): `422` + `_CommentForm` з повідомленням про помилку (через `HX-Reswap` + `HX-Retarget`)
- AntiForgery: через `htmx:configRequest` глобально

**Завдання 1.2.** Додайте `HX-Trigger: commentAdded` до успішної відповіді. Нехай лічильник коментарів у навбарі `<span id="comments-count" hx-get="/comments/count" hx-trigger="commentAdded from:body">` автоматично оновлюється.

### Рівень 2 — Логіка

**Завдання 2.1.** Реалізуйте **модальне вікно через HTMX** без JavaScript-бібліотек:
- Кнопка `<button hx-get="/article/create-modal" hx-target="#modal-container">` завантажує HTML Bootstrap-модалі з сервера
- Controller повертає Partial View з `<div class="modal fade show d-block">`
- Після submit форми всередині модалі — `HX-Trigger: "closeModal"` + оновлення списку статей
- На клієнті: `document.addEventListener("closeModal", () => ...)` закриває модаль та очищає `#modal-container`

**Завдання 2.2.** Реалізуйте **сортування колонок таблиці** через HTMX:
- Заголовки таблиці є кнопками: `<th hx-get="/products?sort=name&order=asc" hx-target="#products-table">`
- Controller читає `sort` та `order` з query, повертає `_ProductTable` Partial View
- Активна колонка підсвічується: Controller включає в Partial `hx-swap-oob` фрагмент що оновлює заголовки

### Рівень 3 — Архітектура

**Завдання 3.1.** Побудуйте **Kanban-дошку** через HTMX:
- Три колонки: «До виконання», «В процесі», «Готово»
- Drag-and-drop через `hx-vals` + JavaScript Drag API (мінімальний): при drop — `hx-patch="/task/{id}/move?column=done"`
- Server pivot: кожен рядок `<div id="task-{id}">` має `hx-get="/task/{id}/card"` що повертає картку з оновленим статусом та OOB-оновленням лічильників колонок
- `HX-Trigger: taskMoved` → оновлення загальної статистики у хедері

## Резюме

- **Виявлення HTMX**: заголовок `HX-Request: true` → `Request.Headers.ContainsKey("HX-Request")`. Дозволяє повертати або повну сторінку, або Partial
- **AntiForgery**: форми серіалізують token автоматично. Для non-form запитів — мета-тег + `htmx:configRequest` + `options.HeaderName`
- **Response Headers**: `HX-Redirect` (навігація), `HX-Trigger` (клієнтські події), `HX-Retarget`/`HX-Reswap` (зміна цілі з сервера)
- **Статус 422** — рекомендований для помилок валідації з HTMX: відповідь вставляється як звичайна успішна
- **Live-search**: `hx-trigger="input delay:300ms"` + Partial View → мінімум коду
- **Infinite scroll**: sentinel `<tr>` з `hx-trigger="revealed"` + `hx-swap="outerHTML"` → lazy loading без JavaScript
- **Inline edit**: три endpoint (view row, edit row, update row) → повна редакція без перезавантаження

У наступній статті — **File Upload**: `IFormFile`, валідація MIME/розміру, збереження у wwwroot та поза ним, streaming для великих файлів, `FileResult` та захист завантажень.
