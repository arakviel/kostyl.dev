# Компоненти та повторюваність: @apply, @utility та патерни

## Проблема, яка виникає у кожного

Щойно ви освоїли Tailwind і починаєте будувати реальний проєкт — з'являється питання:

У мене є кнопка: `inline-flex items-center justify-center gap-2 px-5 py-2.5 font-semibold rounded-xl transition-colors bg-indigo-600 hover:bg-indigo-700 text-white`. Вона використовується **40 разів**. Якщо дизайнер скаже змінити `rounded-xl` на `rounded-2xl` — мені оновлювати 40 місць?

Це реальна проблема. І Tailwind має кілька рішень — але кожне підходить для свого контексту.

---

## Рішення 0: Компоненти (найкраще у фреймворках)

Якщо ви використовуєте React, Vue, Svelte, Angular — **перше і краще рішення**: компонент.

```jsx
// React: кнопка — це компонент, а не CSS
function Button({ children, variant = 'primary', ...props }) {
    const variants = {
        primary: 'bg-indigo-600 hover:bg-indigo-700 text-white',
        secondary: 'border-2 border-indigo-600 text-indigo-600 hover:bg-indigo-50',
        ghost: 'text-slate-600 hover:bg-slate-100',
    }
    return (
        <button
            className={`inline-flex items-center justify-center gap-2 px-5 py-2.5 font-semibold rounded-xl transition-colors ${variants[variant]}`}
            {...props}
        >
            {children}
        </button>
    )
}

// Використання — без повторення класів
<Button>Primary</Button>
<Button variant="secondary">Secondary</Button>
<Button variant="ghost">Ghost</Button>
```

Один компонент — одне місце для зміни. `@apply` не потрібний. Ось чому Tailwind так популярний саме в React/Vue-екосистемі.

## Рішення 1: `@apply` — перенести класи у CSS

Якщо ви не використовуєте компонентні фреймворки — `@apply` дозволяє витягти набір Tailwind-класів у CSS-правило:

```css
/* components.css */
@layer components {
    .btn {
        @apply inline-flex items-center justify-center gap-2
               px-5 py-2.5 font-semibold rounded-xl transition-colors;
    }

    .btn-primary {
        @apply bg-indigo-600 hover:bg-indigo-700 text-white;
    }

    .btn-secondary {
        @apply border-2 border-indigo-600 text-indigo-600 hover:bg-indigo-50;
    }
}
```

```html
<!-- Тепер HTML чистий -->
<button class="btn btn-primary">Primary</button>
<button class="btn btn-secondary">Secondary</button>
```

### Обмеження `@apply`

`@apply` не підтримує **варіанти** повністю. Запис `@apply hover:bg-indigo-700` — компілюється, але рекомендується уникати. Краще виносити тільки базові стилі:

```css
/* Добре: базові стилі в @apply */
.card {
    @apply bg-white rounded-xl border border-slate-200 shadow-sm p-6;
}

/* Уникайте: варіанти в @apply — краще в HTML */
/* hover:, focus:, dark:, responsive: → залишайте у class="" */
```

### Коли НЕ варто @apply

::caution
`@apply` — це компроміс. Ви повертаєтесь до "CSS-файлу з класами", але тепер вони написані через Tailwind-синтаксис. Всі переваги Tailwind (читабельність у HTML, локальність стилів) частково зникають.

Використовуйте `@apply` тільки для реально повторюваних базових патернів: кнопки, inputs, картки. Не застосовуйте до unікальних секцій сторінки.
::

## Рішення 2: `@utility` — кастомний utility-клас (НОВА v4)

**`@utility`** — нова директива Tailwind v4, що дозволяє визначити власний utility-клас **зі всіма можливостями варіантів** (`hover:`, `dark:`, `responsive:`, `group-hover:` тощо).

```css
/* Ваш utility-клас, що поводиться як вбудований Tailwind-клас */
@utility btn {
    display: inline-flex;
    align-items: center;
    justify-content: center;
    gap: 0.5rem;
    padding: 0.625rem 1.25rem;
    font-weight: 600;
    border-radius: 0.75rem;
    transition-property: color, background-color, border-color;
    transition-duration: 150ms;
}

@utility input-base {
    width: 100%;
    padding: 0.5rem 0.75rem;
    border: 1px solid oklch(0.88 0.01 270);
    border-radius: 0.5rem;
    font-size: 0.875rem;
    outline: none;
    transition:
        border-color 150ms,
        box-shadow 150ms;
}
```

**Ключова різниця від `@apply`**: з `@utility` варіанти працюють:

```html
<!-- @utility btn підтримує всі варіанти: hover:, dark:, group-hover:, md: -->
<button class="btn hover:scale-[1.02] bg-indigo-600 text-white dark:bg-indigo-500">
    <button class="btn group-hover:shadow-lg border border-slate-300"></button>
</button>
```

З `@layer components` (старий підхід) варіанти з кастомними класами не завжди поводяться коректно. `@utility` — правильний moderne підхід.

## `@layer base` — глобальні стилі

`@layer base` — найнижчий рівень: скидання браузерних стилів та глобальні налаштування:

```css
@layer base {
    *,
    *::before,
    *::after {
        box-sizing: border-box;
    }

    html {
        scroll-behavior: smooth;
        -webkit-text-size-adjust: 100%;
    }

    body {
        @apply font-sans antialiased text-slate-700 bg-white;
        line-height: 1.6;
    }

    h1,
    h2,
    h3,
    h4,
    h5,
    h6 {
        font-weight: 700;
        line-height: 1.2;
        @apply text-slate-900;
    }

    a {
        @apply text-indigo-600 hover:text-indigo-700 transition-colors;
    }

    img {
        max-width: 100%;
        display: block;
    }

    /* Focus visible — доступність */
    :focus-visible {
        outline: 2px solid oklch(0.6 0.22 270);
        outline-offset: 2px;
    }
}
```

## `@custom-variant`: власні варіанти замість плагінів (НОВА v4)

Одна з найважливіших новинок Tailwind v4 — **`@custom-variant`**. Вона дозволяє створювати власні варіанти (аналоги `hover:`, `dark:`, `focus:`) прямо у CSS, без жодного JavaScript.

У v3 для цього потрібен був JavaScript-плагін:

```javascript
// v3: JS-плагін для кастомного варіанта — застарілий підхід!
plugin(function({ addVariant }) {
    addVariant('hocus', ['&:hover', '&:focus'])
    addVariant('dark', '.dark &')
    addVariant('theme-ocean', '.theme-ocean &')
})
```

У v4 — чистий CSS:

```css
/* v4: @custom-variant — сучасний підхід */
@custom-variant hocus (&:hover, &:focus);
@custom-variant dark (&:is(.dark *));
@custom-variant theme-ocean (&:is(.theme-ocean *));
```

### Синтаксис `@custom-variant`

Є два синтаксиси: **однорядковий** (для простих умов) та **блоковий** (для складних):

**Однорядковий:**

```css
/* @custom-variant {ім'я} {CSS-селектор-умова}; */
@custom-variant hocus (&:hover, &:focus);
@custom-variant focus-within-visible (&:focus-within:not(:focus-visible));
@custom-variant supports-grid (@supports (display: grid));
```

**Блоковий (для медіазапитів та @supports):**

```css
/* @custom-variant {ім'я} { @{медіа-або-умова} { ... } } */
@custom-variant print {
    @media print {
        @slot;
    }
}

@custom-variant motion-ok {
    @media (prefers-reduced-motion: no-preference) {
        @slot;
    }
}

@custom-variant landscape {
    @media (orientation: landscape) {
        @slot;
    }
}
```

`@slot` — спеціальна директива, що позначає місце де буде підставлено CSS-правило.

### Практичні приклади `@custom-variant`

**Приклад 1: `dark` режим через клас на `<html>`**

```css [src/styles/main.css]
@import 'tailwindcss';

/* Dark режим: коли html має клас .dark */
@custom-variant dark (&:is(.dark *));
```

```html
<!-- Тепер dark: варіант реагує на клас .dark на будь-якому предку -->
<html class="dark">
    <body>
        <div class="bg-white dark:bg-slate-900 text-slate-900 dark:text-white">
            Текст адаптується до dark режиму
        </div>
    </body>
</html>
```

**Приклад 2: Multi-theme система**

```css [src/styles/main.css]
@import 'tailwindcss';

/* Три теми: клас на body або html */
@custom-variant theme-ocean  (&:is(.theme-ocean *));
@custom-variant theme-forest (&:is(.theme-forest *));
@custom-variant theme-sunset (&:is(.theme-sunset *));
```

```html
<html class="theme-ocean">
    <div class="
        bg-blue-50
        theme-ocean:bg-cyan-50
        theme-forest:bg-emerald-50
        theme-sunset:bg-orange-50
        text-slate-900
        theme-ocean:text-cyan-900
        theme-forest:text-emerald-900
    ">
        Контент із темою
    </div>
</html>
```

**Приклад 3: `hocus` — hover OR focus одночасно (UX-патерн)**

```css
@custom-variant hocus (&:hover, &:focus-visible);
```

```html
<!-- Замість двох варіантів — один hocus: -->
<!-- До: hover:text-indigo-600 focus-visible:text-indigo-600 -->
<!-- Після: hocus:text-indigo-600 -->
<a class="text-slate-700 hocus:text-indigo-600 transition-colors">
    Посилання
</a>

<button class="bg-slate-100 hocus:bg-slate-200 rounded-lg px-4 py-2 transition-colors">
    Кнопка
</button>
```

**Приклад 4: `print` — стилі для друку**

```css
@custom-variant print {
    @media print {
        @slot;
    }
}
```

```html
<!-- Сховати елементи при друку -->
<nav class="print:hidden">...</nav>
<button class="print:hidden">Підписатись</button>

<!-- Змінити розміри тексту для друку -->
<h1 class="text-4xl print:text-2xl">Заголовок</h1>

<!-- Прибрати тіні та градієнти -->
<div class="shadow-xl print:shadow-none bg-gradient-to-br print:bg-white from-indigo-50">
```

**Приклад 5: Кастомний `data-*` варіант**

```css
/* Варіант для data-атрибуту */
@custom-variant data-active (&[data-active]);
@custom-variant data-loading (&[data-loading]);
@custom-variant data-error (&[data-error]);
```

```html
<!-- Стилізація через data-атрибути без JS className toggle -->
<div data-active
     class="border-2 border-slate-200 data-active:border-indigo-500
            bg-white data-active:bg-indigo-50
            data-active:shadow-lg transition-all">
    Активний стан
</div>

<input data-error
       class="border-slate-300 data-error:border-red-500
              data-error:bg-red-50/50
              focus:outline-none focus:ring-2 data-error:ring-red-500/20
              data-error:focus:ring-red-500/20">

<button data-loading

    <span class="data-loading:hidden">Надіслати</span>
    <span class="hidden data-loading:inline-flex items-center gap-2">

```

        <div class="flex gap-1 p-1 bg-slate-100 rounded-lg" id="tabs">
            <button onclick="setTab(this)" data-active
                    class="flex-1 py-1.5 rounded-md text-sm font-semibold transition-all
                           text-slate-500 [&[data-active]]:bg-white [&[data-active]]:text-indigo-600
                           [&[data-active]]:shadow-sm">
                Головна
            </button>
            <button onclick="setTab(this)"
                    class="flex-1 py-1.5 rounded-md text-sm font-semibold transition-all
                           text-slate-500 [&[data-active]]:bg-white [&[data-active]]:text-indigo-600
                           [&[data-active]]:shadow-sm">
                Аналітика
            </button>
            <button onclick="setTab(this)"
                    class="flex-1 py-1.5 rounded-md text-sm font-semibold transition-all
                           text-slate-500 [&[data-active]]:bg-white [&[data-active]]:text-indigo-600
                           [&[data-active]]:shadow-sm">
                Налаштування
            </button>
        </div>
    </div>

</div>

    el.closest('#tabs').querySelectorAll('button').forEach(b => b.removeAttribute('data-active'))
    el.setAttribute('data-active', '')
}
</script>
</body>

## `@plugin` — офіційні та власні плагіни (v4)

Для підключення офіційних Tailwind-плагінів або власних модулів використовується `@plugin`:

### Офіційні плагіни

```css [src/styles/main.css]
@import 'tailwindcss';

/* Офіційні плагіни через @plugin */
@plugin "@tailwindcss/typography";    /* prose, prose-lg... */
@plugin "@tailwindcss/forms";         /* кращі стилі форм */
@plugin "@tailwindcss/aspect-ratio";  /* aspect-w-*, aspect-h-* */
```

```bash
npm install @tailwindcss/typography @tailwindcss/forms --save-dev
```

```html
<!-- Після @plugin "@tailwindcss/typography" -->
<article class="prose prose-slate prose-lg max-w-prose">
    <h1>Заголовок статті</h1>
    <p>Текст автоматично стилізований через prose...</p>
    <blockquote>Цитата з правильними відступами</blockquote>
</article>
```

### Власний JS-плагін (коли `@custom-variant` недостатньо)

Якщо вам потрібна **динамічна логіка** (обчислювані значення, тема-залежні стилі) — можна писати JS-плагін:

```css
@plugin "./plugins/custom-forms.js";
```

```javascript [plugins/custom-forms.js]
import plugin from 'tailwindcss/plugin'

export default plugin(function ({ addUtilities }) {
    // @custom-variant кращий для простих умов
    // JS-плагін — для складної логіки
    addUtilities({
        '.scrollbar-hide': {
            '-ms-overflow-style': 'none',
            'scrollbar-width': 'none',
            '&::-webkit-scrollbar': { display: 'none' },
        },
        '.text-balance': {
            'text-wrap': 'balance',
        },
        '.font-smoothing': {
            '-webkit-font-smoothing': 'antialiased',
            '-moz-osx-font-smoothing': 'grayscale',
        },
    })
})
```

::tip
**Коли що використовувати:**
- `@custom-variant` → прості CSS-умови (`:hover`, `.dark`, `[data-*]`, `@media`)
- `@utility` → власні атомарні утиліти з CSS-властивостями
- `@layer components` → семантичні класи-компоненти через `@apply`
- JS `@plugin` → складна логіка, що потребує обчислень або теми
::
