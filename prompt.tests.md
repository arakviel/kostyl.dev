# Промпт для генерації тестів (Wayground)

## Мета

Проаналізувати наданий навчальний матеріал і на його основі згенерувати інтерактивний тест для платформи [Wayground](https://wayground.com/). Головний результат твоєї роботи — це генерація `.xlsx` файлу з готовими запитаннями для подальшого імпорту в систему.

## Настанови щодо створення тестів

1. **Глибина запитань**: Запитання мають перевіряти не лише запам'ятовування фактів, але й глибоке розуміння та вміння застосовувати знання на практиці. Уникайте занадто банальних питань.
2. **Типи запитань**: Формуйте різноманітні запитання:
    - `Multiple Choice` (одна правильна відповідь)
    - `Checkbox` (декілька правильних відповідей)
    - `Fill-in-the-Blank` (вставити пропущене слово)
    - `Open-Ended` (відкрите запитання)
3. **Кількість**: За замовчуванням створюйте 10-15 запитань (або іншу кількість, якщо специфічно вказано користувачем), що рівномірно покривають ключові концепції всього матеріалу.
4. **Пояснення (Answer explanation)**: Для кожного запитання **обов'язково** додавайте детальне та зрозуміле пояснення того, чому саме ця відповідь є правильною. Це ключовий елемент навчання.

## Формат виводу (.xlsx файл)

Як фінальний крок своєї роботи, ти **обов'язково повинен автоматично згенерувати та зберегти `.xlsx` файл** у проекті (наприклад, написавши та виконавши Python-скрипт з використанням бібліотеки типу `pandas` або `openpyxl`).

- **Назва файлу**: Має точно збігатися з назвою файлу матеріалу, на основі якого створюється тест, але з розширенням `.xlsx`. Наприклад, якщо тест робиться по матеріалу `01.react-basics.md`, то згенерований файл має називатися `01.react-basics.xlsx`.
- **Місце збереження**: Зберігай згенерований файл у спеціальній папці `tests/` у корені проєкту (бажано відтворюючи в ній структуру папок оригінальних матеріалів, щоб уникнути плутанини).

### Структура колонок файлу `.xlsx`

Файл має суворо відповідати наступній структурі (перший рядок — заголовки колонок, точно як у зразку Quizizz, який підтримується платформою):

1. `Question Text`: Текст запитання. (Обов'язково)
2. `Question Type`: Тип запитання (`Multiple Choice`, `Checkbox`, `Fill-in-the-Blank`, `Open-Ended`, `Poll`, `Draw`). За замовчуванням `Multiple Choice`. (Обов'язково)
3. `Option 1`: Текст 1-го варіанта відповіді (Обов'язково для всіх, крім `Open-Ended` та `Draw`).
4. `Option 2`: Текст 2-го варіанта відповіді (Обов'язково для всіх, крім `Open-Ended` та `Draw`).
5. `Option 3`: Текст 3-го варіанта відповіді (Необов'язково).
6. `Option 4`: Текст 4-го варіанта відповіді (Необов'язково).
7. `Option 5`: Текст 5-го варіанта відповіді (Необов'язково).
8. `Correct Answer`: Правильна відповідь. Вказується номер варіанта від `1` до `5`. Для `Checkbox` — номери через кому (наприклад, `1,2,3`). Залиште поле порожнім для `Open-Ended`, `Poll`, `Draw`, `Fill-in-the-Blank`.
9. `Time in seconds`: Час на відповідь у секундах (наприклад, `30`, `45`, `60`, `120`). За замовчуванням `30`.
10. `Image Link`: Посилання на зображення до запитання (Необов'язково).
11. `Answer explanation`: Детальне пояснення правильної відповіді. (Необов'язково технічно, але **обов'язково** відповідно до вимог промпту для навчальних цілей).

## Твій алгоритм дій

1. Прочитай і проаналізуй наданий користувачем навчальний матеріал.
2. Сформуй якісні запитання, варіанти відповідей, визнач правильні та напиши детальні пояснення.
3. Напиши та автоматично виконай скрипт для створення `.xlsx` файлу за описаною вище структурою колонок.
4. Після успішного створення файлу, виведи коротке повідомлення користувачу із вказанням шляху до збереженого файлу та кількості згенерованих питань.
