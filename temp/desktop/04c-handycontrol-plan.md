## Особливості написання

### Text-First принцип
- 80% тексту, 20% коду
- Кожен контрол спочатку описується текстом
- Потім приклад коду
- Потім пояснення ключових моментів

### Організація контролів
- Групування за функціональністю
- Не просто список, а логічна структура
- Порівняння з аналогами

### Приклади коду
- Для кожного контролу мінімум 1 приклад
- Для складних контролів — 2-3 приклади
- Повні приклади для практичних сценаріїв

### Візуалізація
- Таблиці для порівняння
- Списки для переліку контролів
- Card-group для переваг/недоліків

### Мова
- Українська
- Академічний стиль
- Пояснення "чому", а не тільки "як"

## Питання для уточнення

1. **Розбити на 2 частини чи одна стаття?**
   - Одна велика стаття (10-12k слів) — довідник
   - Дві частини (по 5-6k слів) — легше читати

2. **Рівень деталізації контролів**:
   - Всі 80+ контролів детально (дуже довго)
   - Топ-30 контролів детально + решта коротко (рекомендовано)
   - Тільки унікальні контроли детально

3. **Практичні приклади**:
   - 1 великий приклад (Dashboard)
   - 3 середніх приклади (Dashboard, Chat, Admin)
   - Багато маленьких прикладів для кожного контролу

**Рекомендація**: 
- Одна стаття (довідник)
- Топ-30 контролів детально + решта коротко
- 3 середніх практичних приклади
- Загальний обсяг ~10,000 слів
