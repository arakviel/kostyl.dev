## Що таке HTML? Історія створення

HTML (з англ. _HyperText Markup Language_ — «мова гіпертекстової розмітки») — є стандартом для документів, що передаються в мережі Інтернет. HTML-розмітка інтерпретується браузерами у вигляді відформатованого тексту і може відображатися на екрані монітора комп'ютера, планшета або смартфона.

HTML базується на основі SGML (_Standard Generalized Markup Language_ — стандартної узагальненої мови розмітки), яка з'явилася в 1986 р., і відповідає міжнародному стандарту ISO 8879. Що стосується самого SGML, то він був спочатку призначений для структурної розмітки тексту, але не містив опис зовнішнього виду документа, який у ньому можна було б створити.

Тобто, при тому, що SGML мав на увазі опис синтаксису для написання головних елементів розмітки текстів, причому вже тоді вони отримали назву «тегів», він сам не був системою для розмітки тексту і не мав списку структурних елементів мови для того, щоб використовувати їх при створенні документа.

Але потреба у створенні гіпертекстової мови була, причому вона повинна була мати:

-   опис елементів та випадків їх застосування;
-   перелік елементів документа, який можуть відображати спеціальні програми.

Саме тому в 1991 році Європейська організація з ядерних досліджень (CERN) у Женеві, Швейцарія оголосив про необхідність розробки механізму, що дозволяє передавати гіпертекстову інформацію через Глобальну мережу. За розробку цієї мови взявся співробітник інституту Тім Бернес-Лі. І саме стандарт SGML був основою майбутньої мови HTML (_Hyper Text Markup Language_).

Відповідно до вимог і потреб CERN Тім Бернес-Лі розробляв HTML насамперед для обміну науковою та технічною документацією. Причому цю мову мали без особливих проблем використовувати люди, які не є фахівцями в галузі верстки. Було визначено невеликий набір структурних та семантичних елементів — дескрипторів, які потім стали називати тегами, за допомогою яких створювали досить прості та водночас гарно оформлені документи. Таким чином, HTML успішно вирішував проблеми, що залишилися від SGML.

Рисунок 1. Тім Бернес-Лі

Працюючи в CERN, Тім Бернерс-Лі займався не лише розвитком HTML. До його завдань входила побудова внутрішньої мережі організації. Концепції, які вона реалізувала, були доопрацьовані та переросли у проєкт під назвою «Всесвітня павутина».

Проєкт розумів, що можна буде публікувати документи, розмічені за допомогою HTML, у відкритому доступі. Усі документи мають бути з гіперпосиланнями один на одного, що дозволяло зв'язати їх між собою, перетворюючи їх на формацію, схожу на павутину.

Для реалізації своєї ідеї Бернерс-Лі створив спеціальні програми: HTTP-сервер та WEB-браузер. Перший у світі веб-сайт був розміщений 6 серпня 1991 за [посиланням](http://info.cern.ch/) (ви можете подивитися на його архівну версію [тут](https://www.w3.org/History/19921103-hypertext/hypertext/WWW/News/9201.html)). У його змісті описується принцип роботи мережі, як встановити веб-сервер та створити просту сторінку.

### Стандарти HTML

У 1993 р. з'явився HTML 1.2, який містив лише 40 тегів. Але повноцінного оформлення сторінки вони ще не давали.

Для відображення HTML-сторінок було розроблено спеціальну програму, яка отримала назву «браузер». І першим браузером став Mosaic (рис. 1), який був розроблений в Національному центрі суперкомп'ютерних технологій у США (_National Center for Supercomputer Applications_ — NCSA). За перший рік було встановлено близько двох мільйонів копій програми. Вона підтримувала відображення картинок, розповсюджувалася безкоштовно і містилась на одній дискеті.

Рисунок 2

Наступним етапом розвитку вже HTML-стандарту було створення у квітні 1994 року Консорціуму W3C (_World Wide Web Consortium_).

Оскільки офіційної специфікації HTML 1.0 немає, саме W3C почав займатися підготовкою специфікації HTML наступної версії. Їй одразу дали номер 2.0, щоб підкреслити відмінність від інших версій HTML. Перший результат вдалося отримати через рік насиченої роботи — 1995 року. Серед значних доповнень слід зазначити створення механізму форм для того щоб відсилати інформацію з комп'ютера користувача на сервер.

Паралельно з розробкою HTML 2.0 велася розробка третьої версії стандарту, яка з'явилася в березні 1995 року і містила в собі теги для створення:

-   математичних формул;
-   сторінок;
-   приміток;
-   вставку зображень, які знаходяться в середні тексту;
-   підтримку формату GIF і т.д.

При тому, що цей стандарт був сумісний з другою версією, його реалізація була складна для браузерів того часу.

Крім того, у 1995 році вже існувала потреба у більшому візуальному розмаїтті сторінок. Коштів, пропонованих HTML, особливо у межах стандарту SGML, було недостатньо. Тоді корпорація W3С розпочала створення додаткової системи, що не суперечить основам HTML, але дозволяє описувати візуальне оформлення документів. В результаті з'явився CSS (_Cascading Style Sheets_ — Каскадні таблиці стилів), які являли собою ієрархічні стильові специфікації. Вони мали власний синтаксис, структуру та завдання, які дозволяли доповнити теги HTML візуальним форматуванням. Створення CSS було великим кроком уперед, оскільки потреби у візуальній презентації сторінок значно зросли, а HTML не пропонував при цьому жодних коштів.

### Війна браузерів

До речі, Mozaic вже не був єдиним браузером. З 1994 року у мережі з'явився Netscape Navigator та Opera, а з 1995 року — Internet Explorer. Щоб мати якомога більшу частину потенційної аудиторії, залучити максимальну кількість нових користувачів, Netscape вводила в HTML все нові і нові удосконалення, які підтримувалися, лише однойменним браузером. Практично всі нові теги були спрямовані на покращення зовнішнього вигляду документа та розширення можливостей його форматування.

Влітку 1996 р. було випущено версію Internet Explorer 3.0. Вона підтримувала практично всі розширення Netscape і мала симпатичний і зручний інтерфейс. Тому дуже швидко укорінилася як «другий головний браузер».

Четверті версії обох браузерів вийшли практично одночасно і не відрізнялися одна від одної особливою швидкістю роботи чи іншими параметрами. Якщо Netscape потрібно було купувати, Internet Explorer почав поставлятися безкоштовно в операційній системі Windows і став фактично стандартом галузі. 90-ті роки ХХ століття ознаменувалися таким поняттям як «Війна браузерів», яке, насправді, означає протистояння на ринку Інтернету між Netscape Navigator та Internet Explorer. Ця війна стала проблемою верстальників, оскільки кожен браузер намагався зробити свій внесок у розвиток мови HTML і не слідував стандартам W3C. Тому нерідко на сторінках сайтів можна було знайти запис «Коректне відображення у браузері…» з посиланням на завантаження відповідної програми.

Потрібно відзначити, що наприкінці 1990-х — на початку 2000-х років за рахунок того, що IE входив до Windows за замовчуванням, він став найпопулярнішим браузером, особливо для не дуже досвідчених користувачів. Тож свого часу корпорація Microsoft захопила більшу частку переглядів сайтів в Інтернеті, подавивши Netscape Navigator, хоча згодом свою першість утримати не змогла.

З Netscape Navigator згодом народився Mozilla FireFox, який підтримував стандарти W3C, мав вкладки, а не вікна та ряд фішок, що переманили на свій бік просунутих користувачів та верстальників. Досить багато шанувальників з'явилося й у браузера Opera. 11 грудня 2008 року було випущено браузер Google Chrome, вихідний код якого став доступний під назвою Chromium. З цієї миті будь-яка компанія могла зробити свій браузер на основі цього коду. Наприклад, з'явилися Yandex-браузер або Amigo. Крім того, Opera теж перейшла на двигун WebKit, на якому було створено Chrome.

Chrome повільно, але впевнено займав місце на ринку браузерів, і зараз є переможцем, оскільки його використовують близько 50% користувачів для серфінгу в Інтернеті. Крім того, Google Chrome надає зручні інструменти для розробників вихідного коду — HTML-верстальників, JavaScript-програмістів, тому ми з вами будемо його використовувати в першу чергу.

Слід зазначити, що інструменти розробника зараз присутні у будь-якому браузері, у тому числі й у Internet Explorer останніх версій. І не останню роль тут відіграло доповнення для FireFox під назвою FireBug, яке надавало інформацію про HTML-елементи, CSS-правила для них і дозволяло налагоджувати код JavaScript. В Opera для тих самих цілей було розширення Dragonfly.

Зупинимося ще на понятті «кросбраузерність». Воно має на увазі однакове відображення сайту у всіх браузерах. На даний момент ця проблема стоїть вже не так гостро, як у 2005-2012 рр., оскільки всі сучасні браузери намагаються підтримувати стандарти W3C. ### Стандарти HTML (продовження)

Але повернемось до історії стандартів HTML.

Версія HTML 3.1 офіційно ніколи не пропонувалася, тому наступною версією стандарту HTML стала 3.2, яка була випущена 14 січня 1997 року. У ній було опущено багато нововведень з версії 3.0. Натомість були додані нестандартні елементи, що підтримуються браузерами Netscape Navigator та Mosaic.

18 грудня 1997 року було прийнято четверту версію HTML. Стандарт HTML 4.0, як і стандарт третьої версії, містив багато елементів, специфічних окремих браузерів. Слід зазначити, що у 4-ій версії багато елементів було позначено як застарілі та не рекомендовані до використання. Натомість рекомендовано використовувати таблиці стилів CSS.

Далі йде стандарт HTML 4.01, який було затверджено 24 грудня 1999 року. У ньому були прийняті значні зміни, і цей стандарт був популярний у мережі досить довго.

Паралельно з розробкою стандарту HTML ведеться також розробка альтернативного стандарту XHTML (_Extensible Hypertext Markup Language_) — мова розмітки розширеного гіпертексту, який використовує підхід XML. У ньому пред'являються більш суворі вимоги щодо синтаксису, ніж HTML. XHTML 1.0 був затверджений 26 січня 2000 року за рекомендацією W3C. Варіант XHTML 1.1 схвалений за рекомендацією консорціуму 31 травня 2001 року.

Розробка нових стандартів HTML практично ніколи не припинялася, тому що вимоги до відображення сторінок постійно зростають. З 2007 року розроблявся стандарт HTML5, остаточно затверджений 28 жовтня 2014 року. А з 17 грудня 2012 року вже розробляється стандарт HTML 5.1.

Ми з вами вивчатимемо версію HTML5. Цей стандарт, в порівнянні з попередніми, додав безліч нових тегів: <canvas>, <nav>, <section>, <header>, <footer>, <main> і т.д., нових елементів форм, які розширили можливості щодо структурування HTML-документа.

Крім того, ми будемо використовувати стандарт CSS3 для візуального оформлення сторінок, в якому також додали багато нових властивостей, що полегшують процес створення красивих веб-сторінок без використання картинок або JS-коду.

Слід зазначити, що багато CSS-властивостей, які ми будемо використовувати, відносяться до попередньої версії — CSS 2.1. Але це зрозуміло — CSS3 стала спадкоємицею попереднього стандарту, доповнивши та розширивши його можливості.

### Базові визначення HTML

Створення HTML-документа передбачає розмітку всього вмісту у вигляді тегів (перша назва — дескрипторів), які записуються в кутових дужках. Текст повинен розташовуватися між відкриваючим (початковим) та закриваючим (кінцевим) тегами. Наприклад, теги абзацу з текстом виглядають так:

```html
&lt;p&gt; Текст абзацу &lt;/p&gt;
```

![](html/copy-svgrepo-com.svg)

Існує ще одне поняття — це поняття елемента. Весь текст, укладений між початковим і кінцевим тегом, включаючи самі теги, називається елементом. Сам текст між тегами є змістом елемента. Враховуйте, що зміст елемента може включати будь-який текст, у тому числі й інші елементи.

Для будь-якого тега можна встановити додаткові властивості, які називаються атрибутами. Розміщуються вони тільки в тег, що відкриває, і являють собою пари виду «властивість = "значення "», розділені пробілами.

```html
&lt;p id = "test" class = "par"&gt; Текст абзацу &lt;/p&gt;
```

![](html/copy-svgrepo-com.svg)

HTML є реєстронезалежною мовою розмітки, тому припустимо писати теги як у верхньому, так і в нижньому регістрі. Проте правилом де-факто стало вже давно написання тегів тільки в нижньому регістрі.

Документи HTML є текстовими документами. Вони мають розширення _.html_ або _.htm_, і для їх перегляду необхідний браузер, а для зміни — редактор коду, про вибір якого ми поговоримо нижче.

Ще однією особливістю HTML є ігнорування пробілів та переносів рядків. Як би ви не відформатували текст у HTML-документі ДО додавання тегів — з відступами, зробленими клавішею TAB, з перенесенням кожного рядка (клавіша ENTER) — в браузері він буде просто суцільним текстом.

## Структура HTML-файлу. DOCTYPE

Будь-який HTML-файл має базову структуру, що складається з тегів html, head та body. Але починається вона завжди з оголошення типу документа — DOCTYPE.

Ми розглядаємо синтаксис останнього на даний момент стандарту HTML — це HTML5. Для нього тип документа вказується дуже просто:

```html
&lt;!DOCTYPE html&gt;
```

![](html/copy-svgrepo-com.svg)

Попередніми стандартами є HTML4.01 і XHTML 1.0, які існували в різних варіантах: суворому, перехідному і для фреймів, тому DOCTYPE необхідно було оголошувати по-різному в залежності від типу розмітки, яка використовувалася в HTML-документі.

### Суворий тип документа

Застосовується суворий синтаксис мови відповідного стандарту, а також допускається включати всі теги та атрибути, крім застарілих.

Для HTML 4.01:

```html
&lt;!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" 
      "http://www.w3.org/TR/html4/strict.dtd"&gt;
```

![](html/copy-svgrepo-com.svg)

Для XHTML 1.0:

```html
&lt;!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN" 
      "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd"&gt;
```

![](html/copy-svgrepo-com.svg)

Синтаксис мови залежать від версії, але в обох варіантах треба дотримуватися наступних правил.

Вкрай небажано використовувати такі теги: <applet>, <basefont>, <center>, <dir>, <font>, <isindex>, <noframes>, <plaintext>, <s>, <strike>, <u>, <xmp>, оскільки вони відносяться до застарілих (deprecated). Замість них необхідно задавати форматування за допомогою стилів CSS.

Крім того, не можна додавати до <body> будь-який текст, зображення та елементи форм безпосередньо. Всі ці елементи мають знаходитися всередині інших блокових елементів, наприклад <p> або <div>.

Небажане застосування таких атрибутів, як target для посилань (тег <a>), а також start (тег <ol>), type (теги <li>, <ol>, <ul>) та ін.

Також не дозволяється використання фреймів.

### Перехідний тип документа

У цьому випадку використовується «м'який» синтаксис мови, також можна використовувати всі теги та атрибути, включаючи застарілі.

Для HTML 4.01:

```html
&lt;!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" 
       "http://www.w3.org/TR/html4/loose.dtd"&gt;
```

![](html/copy-svgrepo-com.svg)

Для XHTML 1.0:

```html
&lt;!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN" 
       "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd"&gt;
```

![](html/copy-svgrepo-com.svg)

Мета таких перехідних доктайпів полягає у поступовому знайомстві із синтаксисом мови. Він може допомогти на початковому етапі вивчення мови, особливо якщо ви робите це за старими підручниками або посібниками в інтернеті. У документах із перехідним доктайпом можна використовувати атрибут target, який дозволяє відкривати посилання у новому вікні. Також можна використовувати такі теги, як <center> або <font>, які часто використовують більшість початківців HTML-кодерів. Але в цьому DOCTYPE кадри також не дозволені.

Відразу хочу сзауважити щодо перехідного DOCTYPE — якщо у вас є елементи форматування, які саме на даний момент вже застаріли — такий доктайп дозволить створити надійний документ. Але, якщо ви всерйоз вирішили займатися версткою — ніколи не використовуйте теги <center> або <font> — такі CSS-властивості, як text-align: center або font-family, font-size та color дадуть вам значно більше варіантів керування зовнішнім виглядом документа.

### Тип документа для фреймів

Фрейми свого часу (початок 2000-х років) були дуже популярним інструментом для завантаження в один HTML-файл 2-х, 3-х або більше інших HTMLдокументів. Але на даний момент існує безліч способів створити красивий HTML-файл, не вдаючись до таких складнощів. Для фреймової структури — frameset — застосовується спеціальний DOCTYPE, який за синтаксисом подібний до перехідного.

Для HTML 4.01:

```html
&lt;!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Frameset//EN" 
    "http://www.w3.org/TR/html4/frameset.dtd"&gt;
```

![](html/copy-svgrepo-com.svg)

Для XHTML 1.0:

```html
&lt;!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Frameset//EN" 
    "http://www.w3.org/TR/xhtml1/DTD/xhtml1-frameset.dtd"&gt;
```

![](html/copy-svgrepo-com.svg)

У вас може виникнути питання: а навіщо вам знати про всі ці типи документів, якщо зараз достатньо використовувати простий <!DOCTYPE html>?

Справа в тому, що в мережі ще багато сайтів, створених 5-10-15 років тому, для яких, можливо, вам доведеться робити редизайн (тобто зміна зовнішнього вигляду) або просто правити частину коду. І важливо розуміти, що можна, що не можна залишити у документі, щоб код вважався правильним, тобто відповідним до стандартів організації W3C.

### Базова структура HTML-документа

Отже, із DOCTYPE ми розібралися. Тепер розглянемо, які теги є основою HTML-файлу.

```html
&lt;!DOCTYPE html&gt;
&lt;html lang = "en"&gt;
&lt;head&gt;
    &lt;meta charset = "UTF-8"&gt;
    &lt;title&gt; First HTML Document &lt;/title&gt;
&lt;/head&gt;
&lt;body&gt;
    &lt;!-- Тут буде розміщено основний код --&gt;
&lt;/body&gt;
&lt;/html&gt;
```

![](html/copy-svgrepo-com.svg)

Кореневим тегом є <html>. Всі інші теги поміщаються в нього, як матрьошки. Цей тег прийнято вказувати з атрибутом lang, який відповідає за мову, якою створена дана HTML-сторінка. У прикладі lang = "en", оскільки англійська є найбільш розповсюдженою мовою у світі, текст між тегами <title> теж англійською.

Якщо ж ви створюєте сторінку з російським текстом, то значення цього атрибуту має бути «ru»: <html lang="ru">.

Далі в структурі документа йде тег <head>, вміст якого не відображається на сторінці. Проте це важливий тег, що виконує «службові» функції. По-перше, у ньому вказується <title> документа, який відображається на вкладці у браузері.

Рисунок 36

Зараз зауважу лише те, що неправильна вказівка кодування призведе до того, що текст вашої сторінки відображатиметься незрозумілими знаками, чого дуже не хотілося б:

![](html/copy-svgrepo-com.svg)

### Кодування документа

Справа в тому, що в системі Windows за умовчанням для файлів введено кодування символів в системі ASCII (_American Standard Code for Information Interchange_ — американський стандартний код для обміну інформацією), до якої входять латинські та російські символи, цифри, розділові знаки та ін. HTML відповідає кодуванню windows-1251.

ASCII — це однобайтне кодування, яке дозволяє отримати файл меншого розміру, але й за своїми можливостями поступається двобайтним кодуванням UTF-8. Наприклад, вам складно буде створити на ній багатомовний сайт, який має відображатися українською, англійською та, наприклад, чеською мовою. Її можливостей просто не вистачить для відображення символів усіх мов.

У цьому сенсі двобайтне кодування UTF-8 є більш універсальним, оскільки вміщує коди символів на великій кількості мов. Саме тому вона стала найпопулярнішою в даний час і є стандартом «де-факто» у світі web-розробок.

Крім зазначених кодувань для російської мови є ще ISO-8859-5 та KOI-8-R, але їх настільки мало використовують, що немає сенсу розглядати в даному курсі. Якщо ви встановили на комп'ютер програму Notepad++, відкрийте її і в меню Кодування -> Кодування -> Кирилиця перегляньте доступні варіанти.

### Теги заголовків та абзаців

Заголовки — обов'язкова частина HTML-документа. Є навіть два нових теги в HTML5, які без заголовків не пройдуть валідацію — це <article> і <section>. Але їх ми розглядатимемо пізніше.

У HTML передбачено 6 рівнів заголовків, які відрізняються один від одного розміром шрифту. Позначаються заголовки таким чином:

```html
&lt;h1&gt; Заголовок 1-го рівня &lt;/h1&gt;
&lt;h2&gt; Заголовок 2-го рівня &lt;/h2&gt;
&lt;h3&gt; Заголовок 3-го рівня &lt;/h3&gt;
&lt;h4&gt; Заголовок 4-го рівня &lt;/h4&gt;
&lt;h5&gt; Заголовок 5-го рівня &lt;/h5&gt;
&lt;h6&gt; Заголовок 6-го рівня &lt;/h6&gt;
```

![](html/copy-svgrepo-com.svg)

Літера h у тезі означає, що це заголовок (від англійського _heading_), а цифра — який рівень заголовка. Чим більша цифра — тим менша важливість та розмір шрифту заголовка.

Подивимося, як це виглядає в редакторі коду та в браузері:

![Рисунок 46](html/NBZv9FwGZGZKQxGGygMhuFG4W4Oixiac.jpg)

Рисунок 46

Різниця між заголовками різних рівнів, очевидна. <h1> — найбільший і помітний, а <h6> — вже майже нечитається. Власне, і вага цих заголовків в розрізі пошукових систем теж різна. Зазвичай, заголовків типу h1 на сторінці має бути один або два — це назва компанії та назва сторінки. Ще бажано, щоб текст h1, що визначає назву сторінки, збігався із вмістом тега <title> в блоці <head>.

Всі інші заголовки ви можете використовувати на свій розсуд, необов'язково в порядку зростання їх рівня, але бажано, щоб після h1 йшов заголовок h2, оскільки він також важливий для ранжування вашої сторінки.

**Примітка 1.** _Н__а рисунку стрілочкою показана кнопка у вигляді блискавки у верхньому правому кутку Brackets. Це Live Preview (Попередній_ _перегляд) у браузері Google Chrome, «вшитий» у редактор коду. Натискання цієї кнопки під час редагування_ _HTML__\-документа дозволяє в реальному часі переглянути всі зміни, що відбуваються в редакторі. Щоб це побачити,_ _важливо_ _розташувати вікно редактора та вікно браузера поруч, як на рисунку_ _вгорі._

Коли включений Live Preview, в браузері цією рамочкою виділений елемент, на якому зараз розташований курсор (на рисунку це h2). Якщо ви переміщатимете курсор вище або нижче, наприклад, клавішами або , то побачите, як переміщується рамка з одного елемента на інший.

**Примітка 2.** _Я__кщо у Brackets вам доводиться створювати блоки зі схожим або ідентичним змістом,_ _важливо_ _скористатися поєднанням_ _клавіш CTRL+D_ _(від англ. duplicate — дублювати). Якщо курсор знаходиться в будь-якому місці рядка, то буде продубльовано весь рядок. Якщо ви виділите частину слів або кілька рядків, то продублюється весь виділений контент._ _І ц__е дуже зручно._

Що стосується заголовків, то вам слід знати, що всі вони є блоковими елементами, які за замовчуванням в браузерах виділяються жирним шрифтом і мають відступи до і після їхнього тексту.

Потрібно ще розуміти, що не можна всю сторінку заповнити лише заголовками — це буде неправильно з погляду семантики HTML-документу. І пошукачам теж наврядчи сподобається. Вдумайтесь — хіба може книга, стаття у журналі чи газеті складатися з одних заголовків? Звичайно, що ні. Після заголовка повинен обов'язково йти основний текст, який у пресі зазвичай розбивається на абзаци. Розглянемо, як створювати їх в HTML.

### Декілька слів про атрибути

Для будь-якого тега можна задати атрибути, які певною мірою виділяють його серед інших. Атрибути записуються тільки в початковий тег, відокремлюються від назви тега і один від одного пробілами. Загалом це виглядає так:

```html
    Текст елемента 
&lt;/елемент&gt;
```

![](html/copy-svgrepo-com.svg)

Реальний приклад:

```html
&lt;h2 title = 'Стаття про користь вітамінів для дітей' id = 'article1' class = 'article-header'&gt; 
    Вітаміни для дітей 
&lt;/h2&gt;
```

![](html/copy-svgrepo-com.svg)

У цьому прикладі для заголовка 2-го рівня використані універсальні атрибути title, id і class, тобто, такі, які можна додати до будь-якого елемента. Ще до універсальних відносяться tab index і data\-атрибути.

Але також є атрибути, які характерні лише для певних тегів. Наприклад, для вбудовування зображення потрібен тег з атрибутами src та alt, які характерні тільки для нього (хоча src потрібен ще для таких тегів, як <script> та <iframe>):

```html
&lt;img src = 'images/photo.jpg' alt='Картинка'&gt;
```

![](html/copy-svgrepo-com.svg)

При додаванні атрибутів потрібно дотримуватися кількох простих правил:

-   Атрибути відокремлюються один від одного пробілами.
-   Значення атрибутів записують у подвійні або одинарні лапки.
-   Якщо всередині значення атрибуту необхідно використовувати лапки або апостроф, необхідно поєднувати обидва види лапок з урахуванням їх вкладеності:

```html
&lt;p title = 'Абзац із книги Б. Акуніна "Азазель"'&gt;
```

![](html/copy-svgrepo-com.svg)

Тут значення атрибута title взято в одинарні лапки, а всередині використані подвійні назви книги. Цей атрибут додає підказку до елемента, яка з'являється при наведенні на елемент у браузері.

![Рисунок 49](html/0CJvgQnKbyYAuBpAP4F7Ilm3jQ3kNFGE.png)

Рисунок 49

-   Якщо значення атрибута можна подати у вигляді логічної величини true (_істина_) або false (помилка), або інакше «такє» або «ні», то записувати його можна в кількох рівнозначних варіантах:

```html
&lt;input type = "checkbox" checked&gt;
&lt;input type = "checkbox" checked = ""&gt;
&lt;input type = "checkbox" checked = "checked"&gt;
```

![](html/copy-svgrepo-com.svg)

**Примітка.** _Всі браузери налаштовані на максимально правильне відображення тегів та їх атрибутів, тому включають атрибут навіть за наявності в ньому значень_ _на кшталт true або 1, які є неприпустимими. Краще ж уникати таких варіантів,_ _тому що_ _вони суперечать специфікації HTML5._

А тепер поговоримо про застарілі атрибути. Наприклад, атрибут align дозволяє вирівняти текст в абзаці, заголовку або іншому блоковому елементі по лівому або правому краю, по ширині елемента або по центру (align = "left", або align = "right", або align = "justify", або align = "center"). Ви, напевно, знайдете в Інтернеті купу посібників, які рекомендують використовувати саме цей атрибут для вирівнювання тексту.

Тільки ось зі стандартом HTML5 цей атрибут несумісний. І валідатор видасть таке повідомлення: «Атрибут align для елемента p застарів. Замість нього використовуйте CSS»

### Що таке Lorem Ipsum?

Справа в тому, що далеко не завжди у вашій практиці доведеться форматувати готовий текст, наданий замовником. На жаль, замовники з ним досить часто запізнюються.

Крім того, ви можете створювати шаблони сайтів, про реальний зміст яких на етапі розробки можете лише здогадуватися. Тому вам знадобиться шаблонний текст, який починається словами _Lorem ipsum_. Його, до речі, ще називають «текст-риба».

Цитата з Вікіпедії: «Lorem ipsum — класичний варіант умовного беззмістовного тексту, що вставляється в макет сторінки. Використовується як заповнювач принаймні з XVI століття. Тобто це текст, який допомагає заповнити сайт контентом, але при цьому не має сенсу. Це дуже зручно, крім іншого, на етапі навчання. Можна заповнювати за допомогою _Lorem ipsum_ будь-які тестові елементи (ми вже знаємо про заголовки та абзаци), не замислюючись про кількість та зміст тексту.

Докладніше про абревіатури:

Важливо розуміти, що всередині абревіатури не повинно бути прогалин (виняток — текст усередині фігурних дужок). А наприкінці обов'язково потрібно натиснути клавішу Tab, щоб розкрити абревіатуру і отримати відформатований за правилами HTML-текст. Знову ж таки, після тексту абревіатури пробіл ставити не треба, а курсор має знаходитися саме в кінці, а не десь у середині набраного тексту.

Давайте розглянемо дещо складнішу абревіатуру. На цей раз із частково осмисленим текстом:

```html
&lt;h2&gt; Second heading &lt;/h2&gt;
&lt;p&gt; Lorem ipsum dolor sit amet, consectetur adipisicing elit. Placeat corporis quasi perspiciatis? Aperiam eveniet dolorem culpa distinctio, rem quos adipisci. &lt;/p&gt;
&lt;p&gt; Debitis, dolorem! Sunt autem veritatis magnam! Ipsa, dolorem harum laborum praesentium, quas unde ab, alias saepe ullam similique nulla beatae. &lt;/p&gt;
```

### Вкладені теги

Щоб виділити частину тексту жирним або курсивним шрифтом, можна використовувати теги, <b> та <i>:

```html
&lt;p&gt; У понеділок &lt;b&gt;13 травня 1876&lt;/b&gt; року о третій годині, в день по-весняному 
свіжий і по-літньому теплий, &lt;i&gt;в Олександрівському саду&lt;/i&gt;, на очах у численних 
свідків, трапилась жахлива подія, яка не вкладається в жодні рамки. &lt;/p&gt;
```

![](html/copy-svgrepo-com.svg)

Тег <b> походить від слова bold (_жирний_), а <i> — від слова italics (_курсив_), відповідно використання цих тегів призводить до виділення тексту між ними за допомогою жирного або курсивного зображення. Ці елементи належать до групи тегів фізичного форматування.

На відміну від блокових тегів заголовків та абзаців ці теги є рядковими (ще їх називають лінійними від англійського inline).

![Рисунок 57](html/qY-mZIsmAxdWRH5P8W5TXONyE0NmmUji.png)

Рисунок 57

Тому обертають у такі теги не всі речення чи блок тексту, а якусь його частину, наприклад, 2-3-4 слова, як у прикладі на рисунку.

Так само працюють ще 2 теги — <strong> і <em>, тобто, <strong> виділяє текст жирним, а <em> — курсивом, але ці 2 елементи відносяться до групи тегів логічного форматування. У контексті розмітки сторінки виконують роль «підсилювача уваги», тобто, покликані показати (переважно пошукачам, ніж відвідувачам), що виділений у них текст важливий для користувача.

Але теги фізичного, і логічного форматування знаходяться всередині блокових елементів (у прикладі абзац — <p>), тобто, є вкладеними або дочірніми відповідно до абзацу. А от абзац, відповідно до вкладених тегів є батьківським елементом.

Для вкладених тегів важливо дотримуватися «правила матрьошки»: тег, який відкритий найпершим, має бути закритий останнім:

```html
&lt;b&gt;&lt;i&gt; в Олександрівському саду &lt;/i&gt;&lt;/b&gt;
```

![](html/copy-svgrepo-com.svg)

![Рисунок 58](html/rFQwsiUqNQNafe321OIVkQvZ9DJ2Epxb.png)

Рисунок 58

Вкладені теги можуть бути не лише малими. І рівень вкладення може бути досить глибоким. Але це правило все одно залишається в минулому — тег, який відкритий першим, має закриватися останнім.

### Теги div та span

Як ви розумієте, із заголовків та абзаців не побудуєш повноцінну сторінку. Тому в специфікації є ще безліч тегів, з якими ми маємо познайомитися. І одними з елементів, що використовуються, є div-и (від англ. _division_ — розділ). Це блокові елементи, які, на відміну від абзаців не мають відступу зверху і знизу. Це елементи, із яких будується внутрішня структура сайту.

Їх код дуже простий:

```html
&lt;div&gt; текст &lt;/div&gt;
```

![](html/copy-svgrepo-com.svg)

Давайте використаємо просту Emmet-абревіатуру:

```html
div*4&gt;lorem40
```

![](html/copy-svgrepo-com.svg)

Отримаємо такий файл:

![Рисунок 59](html/SHuFKoS2uwJCO6yA0BHTQNxali3v-iOy.png)

Рисунок 59

Візуально тег <div> виглядає не досить гарно — маса тексту без особливого форматування. Але цим він і зручний — для нього можна задати те CSS-форматування, яке потрібне для даної конкретної ситуації. І зазвичай для div-ів задають атрибут class, який дозволяє урізноманітнити зовнішній вигляд цих елементів. Але про це трохи пізніше, коли розглядатимемо стилі CSS.

Також часто всередині тексту використовують теги <span></span> (від англ. _інтервал_). Це малі теги, які призначені для об'єднання невеликої кількості тексту з метою завдання для нього загального форматування. Практично обов'язковим для тега span є атрибут class. Саме він дозволяє зробити форматування для різних <span> різноманітним за допомогою правил CSS.

### Поодинокі теги

У прикладі blockquote всередині цитати був використаний тег <br> (від англ. _break_ — розбивати). Він призначений для перенесення за ним тексту, на наступний рядок і є малим, тобто, зазвичай розміщується у тексті абзаців, div-ів та інших елементів.

Оскільки тег немає внутрішнього вмісту, то закриваючий тег йому не потрібен, тобто, він відноситься до групи одиночних тегів або тегів без вмісту.

До них відноситься тег <img> для вбудовування зображень і ціла група тегів <input>, з яких складаються форми.

Розглянемо ще один такий тег — це <hr>, або горизонтальний рядок (від англ. _horizontal row_).

Власне, він і виводить у браузер горизонтальну лінію. HTML4.01, для якої можна було б задати ряд атрибутів, які в HTML5 є скасованими. Тому всі «прекраси» потрібно ставити через CSS.

У файл _blockquote.html_ можна додати горизонтальні лінії для візуального відокремлення цитат від тексту абзацу. Як видно з рисунка, <hr> — це блоковий тег, який займає весь доступний простір у браузері.

## Правила HTML

1\. Всі недоліки, табуляція та початок нових рядків перетворюються на один великий недолік. Тому дуже бажано форматувати текст із відступами:

```html
&lt;body&gt;
&lt;h1&gt; Заголовок 1 рівня &lt;/h1&gt;
&lt;div id = "test"&gt;
    &lt;p class = "block"&gt; Lorem ipsum dolor sit amet, consectetur adipisicing elit. Odio rerum, praesentium repellat cupiditate dolorum voluptatum. &lt;/p&gt;
    &lt;p class = "block"&gt; Blanditiis similique, modi oficiis, dignissimos asperiores totam quae aut doloribus quam esse reprehenderit quos sed. &lt;/p&gt;
    &lt;p class = "block"&gt; Eum facilis voluptates ut dicta, delectus, tempore animi repudiandae qui quis voluptas maxime ullam, assumenda. &lt;/p&gt;
&lt;/div&gt;
&lt;/body&gt;
```

![](html/copy-svgrepo-com.svg)

2\. Весь текст має бути відформатований у вигляді тегів. Обов'язково мають бути початковий та кінцевий:

```html
&lt;елемент&gt; Текст елемента &lt;/елемент&gt;
```

![](html/copy-svgrepo-com.svg)

3\. Якщо елемент не має вмісту, кінцевий тег йому не потрібен.

```html
&lt;br&gt;
&lt;hr&gt;
```

![](html/copy-svgrepo-com.svg)

4\. Завжди мають бути теги основи: <html>, <head>, <title>, <body>. І вони мають відповідати змісту документа.

5\. Правило «матрьошки»: завжди дотримуватися вкладеності елементів і не допускати їх перетину:

```html
&lt;b&gt;&lt;i&gt; Текст усередині тегів &lt;/i&gt;&lt;/b&gt;
```

![](html/copy-svgrepo-com.svg)

6\. Значення атрибутів завжди беруться у лапки. Атрибутів елемента може бути декілька, вони відокремлюються один від одного пробілами:

```html
&lt;img src = "images/someimage.jpg" alt = "Альтернативний текст" id = "pic"&gt;
```

![](html/copy-svgrepo-com.svg)

7\. Універсальними атрибутами для всіх тегів є: id, class, style, title, tabindex.

```html
&lt;div id = "box" class = "block" title = "Блоковий елемент" style = "font-family: Verdana, Tahoma, sans-serif"&gt; 
     Текст елемента 
&lt;/div&gt;
```

![](html/copy-svgrepo-com.svg)

8\. Коментарі в HTML призначені для того, щоб приховати частину коду та записуються так:

```html
&lt;-- Коментар --&gt;
```

![](html/copy-svgrepo-com.svg)

## Стилі CSS

От ми й дісталися візуального форматування сторінки. З погляду семантики, необхідно відокремлювати «мух від котлет», тобто, візуальне форматування сторінки сайту з його HTML-структурою. А CSS якраз і займається форматуванням. І має масу властивостей для цього у своєму арсеналі.

Існують чотири способи, як додати CSS-форматування:

1.  За допомогою атрибуту style;
2.  Всередині тегів <style> у блоці <head>;
3.  За допомогою тега <link>;
4.  За допомогою директиви @import.

У цьому занятті ми розглянемо перші 2 способи. Інші залишимо для наступних уроків.

### Внутрішні стилі CSS (inline styles)

Спочатку розглянемо, як можна задати форматування для кожного елемента.

Тому призначений атрибут style, про який вже йшлося, є одним із універсальних, і застосовується для будь-якого тегу.

Наприклад, нам необхідно змінити колір і стиль тексту для абзацу, а також призначити для нього інший шрифт. Робиться це так:

```html
&lt;p style = "color: #999; font-style: italic; font-family: 'Open Sans', Verdana, Tahoma, sans-serif"&gt;
```

![](html/copy-svgrepo-com.svg)

Після останнього значення властивості ставити крапку з комою необов'язково. Важливо додати її, якщо ви ще хочете написати одну властивість.

### Стилі для сторінки

Набагато краще навіть на початковому етапі використовувати другий спосіб — форматування всередині сторінки у спеціальних тегах <style></style>, які розміщуються у нижній частині блоку <head>. У цьому випадку стилі використовуються лише для тієї сторінки, на якій вони будуть розміщені. Це якраз прийнятний варіант для початку навчання HTML і CSS — в межах одного документа ви бачите і HTML-розмітку, і CSS-стилі.

Цей вид форматування дозволяє встановити стилі для селекторів CSS — спеціальних описів для елемента або групи елементів, до яких застосовуються певні стильові правила форматування.

На даний момент CSS3 надає нам безліч різних селекторів, з якими ми знайомитимемося послідовно на різних прикладах. Зараз зупинимося на простих селекторах: універсальний селектор, селектор елемента, або тега, селектор групи, селектор класу та селектор id.

Універсальний селектор застосовується для призначення правил одразу для всіх елементів на сторінці, включаючи html і body. Позначається за допомогою зірочки (\* — клавіші SHIFT+8).

Він потрібний для того, щоб встановити однакові правила для всіх елементів сторінки. Наприклад, замість того, щоб задавати розмір шрифту (font-size) та сімейство шрифту (font-family) тільки для абзаців, встановимо їх одразу для всієї сторінки:

### Селектор id

Селектор id має на увазі, що на сторінці для якогось одного елемента вказаний атрибут id — унікальний ідентифікатор (_identifier_). Значення цього атрибута в лапках не може повторюватися в межах однієї сторінки, інакше жодної унікальності не вдасться. Позначається селектор id символом «дієз» # поруч зі значенням атрибута id потрібного елемента. У прикладі нижче ми використовуємо 2 різні id для того, щоб задати різний колір фону різним заголовкам:

```html
#main-header {
    background-color: #adf5ad;
}
#use-technology {
    background-color: #ffeb95;
}
```

![](html/copy-svgrepo-com.svg)

![Рисунок 73](html/G_sWSIuNxlVTMbVppw5RlmGjxPAr7sF6.png)

Рисунок 73

У HTML-розмітку ми додали саме такі id для різних заголовків. Подивитися приклад можна у файлі _page-style.html_.

Зверніть увагу, що значення кожного id представлено в єдиному екземплярі. Це важливо, інакше не можна говорити про унікальність та валідність документа.

## CSS-властивості

Ми вже розглянули частину CSS-властивостей, але не надто детально зупиняючись на їх значеннях. Вважаю, було інтуїтивно зрозуміло, що властивість color відповідає за колір шрифту, а font-size — за розмір шрифту елемента.

Тепер давайте поговоримо про властивості та їх можливі значення докладніше. Відразу зазначу, що в рамках першого заняття ми розглянемо найнеобхідніші. Решту ми розглядатимемо в ході вивчення курсу.

### Варіанти призначення кольору

Для встановлення кольору різних елементів можна використовувати декілька правил. Ми розглянемо сьогодні властивість color, що відповідає за колір тексту, та background-color, що управляє кольором фону елемента.

І в тій, і в іншій властивості можна задавати значення в наступних форматах:

```html
color: blue; /* назва кольору англійською мовою */
color: #ff0000; /* 16-річне значення кольору, що використовує цифри від 0 до 9 та літери від a до f */
color: #f00; /* усічене 16-річне значення для кольорів, у яких однакові пари цифр */
color: rgb (255, 0, 0); /* значення кольору у системіrgb. По кожному каналу r - red, g - green, b - blue значення кольору змінюються від 0 до 255 одиниць*/
color: rgba(255, 0, 0, .5); /* значення кольору системіrgb+ альфа-канал, тобто, можливість задати прозорість для кольору від 0 до 1*/
color: hsl(0, 100%, 50%); /* значення кольору у системіhsl- hue -saturation - lightness */
color: hsla(0, 100%, 50%, .5); /* значення кольору у системіhsl+ альфа-канал, тобто, можливість задати прозорість для кольору від 0 до 1*/
```

![](html/copy-svgrepo-com.svg)

З усіх наведених варіантів найменш зрозумілими, мабуть, будуть два останніх, оскільки колірна система HSL (Hue, _Saturation and Lightness_ — тон, насиченість та світло) навряд чи була предметом вашого вивчення на курсі Photoshop — там вона теж нечасто використовується (для Web, принаймні).

### Властивості шрифту

Англійською шрифт — це font, тому властивості для шрифту в CSS починаються саме з цього слова. Їх декілька, і вони виконують наступні зміни шрифтів на сторінці:

1\. На сімейство шрифтів вказує властивість font-family. У значенні цієї властивості можна перелічити через кому кілька шрифтів у тій послідовності, в якій ви (або замовник) хотіли б, щоб вони відображалися на сайті. Наприклад, як у записі.

```html
body { font-family: Lato, Verdana, Geneva, sans-serif; }
```

![](html/copy-svgrepo-com.svg)

передбачає, що браузер спочатку спробує відобразити текст за допомогою шрифту _Lato_. Але, якщо його немає на комп'ютері користувача, виведе вміст шрифтом _Verdana_. Якщо ж і шрифт _Verdana_ відсутній, буде використано шрифт _Geneva_. І в тому випадку, якщо немає жодного із зазниченихних шрифтів, буде використаний шрифт за замовчуванням, тип якого задається спеціальним ключовим словом (їх ще називають шрифти-псевдоніми):

-   serif — для шрифтів із зарубками (за замовчуванням зазвичай _Times New Roman_);
-   sans-serif — для шрифтів без зарубок або рубаних шрифтів (зазвичай _Arial_)
    
    ![Ovlm7qK7k5DJq2f_Cboc69CxLF88R80c](html/Ovlm7qK7k5DJq2f_Cboc69CxLF88R80c.png)
    
    Рисунок 85
-   monospace — для моноширинних шрифтів, у яких символи мають однакову ширину (зазвичай _Consolas_ або _Courier New_);
-   cursive — для курсивних шрифтів (як варіант — _Comic Sans MS_);
-   fantasy — для фантазійних або декоративних шрифтів (представник — _Impact_).

Прийнято вказувати шрифт-псевдонім останнім, після найбільш відповідних варіантів. Але, якщо вас, наприклад, цілком влаштовує стандартний шрифт без зарубок _Arial_ (а web-сторінки зазвичай використовують рубані шрифти), то ви можете скоротити запис властивості:

```html
body { font-family: sans-serif; }
```

![](html/copy-svgrepo-com.svg)

Для тестів можна використовувати файл _font-family-test.html_.

Подивитися, які шрифти використовуються за замовчуванням, можна налаштувати браузер. Наприклад, для Chrome вони виглядають так:

2\. Стиль шрифту визначається властивістю font-style. Значення: normal italic oblique inherit. Перше значення — normal — це значення за замовчування, тобто, звичайне зображення тексту. Значення italic — це курсивне накреслення, яке передбачає наявність курсивного шрифту в системі користувача, oblique — накреслення шрифту з нахилом, в herit — успадковує значення батьківського елемента.

Абревіатура Emmet: fs, fsi, fso, fsn.

3\. За насиченість шрифту відповідає властивість font-weight, що має значення normal Болд Болдер lighter 100 200 … 800 900 inherit. За замовчуванням використовується значення normal — звичайна насиченість шрифту, яка відповідає файлам шрифту, до яких входить слово Regular. Відповідно значення bold відповідає за напівжирне накреслення. Значення bolder та lighter змінюють жирність шрифту у більшу чи меншу сторону щодо значення цієї властивості у батьківського шрифта. Одиниці від 100 до 900 вказуються з кроком у 100 і розподіляються так: від 100 до 300 — це, як правило, тонкий шрифт, 400-500 — звичайне зображення, від 600 — напівжирне зображення. Якщо розглядати їх докладніше, то зображення в одиницях має такі значення:

-   100 — Thin (Hairline);
-   200 — Extra Light (Ultra Light);
-   300 — Light;
-   400 — Normal;
-   500 — Medium;
-   600 — Semi Bold (Demi Bold);
-   700 — Bold;
-   800 — Extra Bold (Ultra Bold);
-   900 — Black (Heavy).

Справжнє відображення шрифтів на сторінці дуже залежить від того, скільки файлів шрифту складає вибраний шрифт для сторінки або елемента. Наприклад, шрифт _Lato_, який встановлений у мене для дизайнерських цілей, і, швидше за все, відсутній на вашій машині, має таку кількість файлів шрифту:

6\. Висота рядка — інтерліньяж, або міжрядковий інтервал — властивість line-height. Ця властивість не належить безпосередньо до групи fonts, але пов'язана з цими властивостями, оскільки складно уявити добре відформатований текст без вказання висоти рядка. Як значення цієї властивості, можна задати множник значення у будь-яких одиницях відсотки normal inherit. Зазвичай висота рядка задається не менше 120% або множником 1.2, але вона залежить від самого шрифту і обчислюється автоматично (значення normal). Від'ємні значення не допускаються. Найчастіше інтерліньяж вказується у вигляді множника або у відсотках. Переглянути варіанти відображення різних значень цієї властивості можна у файлі _line-height-test.html_. Виняток становить розмір шрифту та висота рядка — вони вказуються через слеш. Синтаксис цієї властивості такий:

```html
font: [font-style font-variant font-weight font-stretch] 
       font-size [/line-height] font-family inherit
```

![](html/copy-svgrepo-com.svg)

Обов'язковими у цій властивості є розмір та сімейство шрифту. Решта всіх властивостей задаються при необхідності в тому порядку, який представлений вище. З властивістю font-stretch вам доведеться познайомитись самостійно, оскільки вона не надто часто використовується.

Слід зазначити, що всі властивості, які ви не вказали, набувають свого значення за замовчуванням.

### Вирівнювання тексту

Властивість text-align необхідна для вирівнювання тексту в блоковому елементі по лівому краю (left — значення за замовчуванням), по правому краю (right), по центру (center) або ширині (justify). І саме його варто використати, коли ви хочете центрувати текст замість застарілого тегу <center>.

```html
text-align: left right центр Justify start end
```

![](html/copy-svgrepo-com.svg)

Нові значення start і end з'явилися у специфікації CSS 3.0 та позначають наступне:

-   start — працює так само, як і left, якщо текст йде ліворуч, і як right, якщо текст йде справа наліво;
-   end — працює так само, як і right, якщо текст йде ліворуч, і як left, якщо текст йде справа наліво;

Протестуйте вирівнювання тексту за допомогою файлу _text-align-test.html_.

### Одиниці вимірювань у CSS

Важливо: будь-які одиниці вимірювань пишуться разом із цифрами. Тобто, не можна поставити цифру, пробіл, а потім вказати одиницю виміру.

Вірно: 16px.

У CSS існують абсолютні та відносні одиниці вимірів.

Що стосується абсолютних одиниць, то CSS можна використовувати звичні нам зі школи знайомі сантиметри (cm) і міліметри (mm), менш звичні дюйми (in), характерні для друкарень пункти (_pt — 1/72 дюйма_) та піки (_pc — 1/6 дюйми_). Але варто зазначити, що у реальній верстці використовуються вони дуже рідко. Проте є ще одна абсолютна одиниця — піксель, або px, — використовується часто, тому що саме в пікселях вимірюється роздільна здатність монітора або іншого екрана; і браузер також багато величин перераховує у пікселях.

А от відносні одиниці використовуються дуже часто. Мабуть, найчастіше використовуваними одиницями є em і rem, які з'явилися нещодавно, а також відсотки (%).

Що між ними є спільного? 1em та 100% — це розмір шрифту батьківського елемента. Тобто, якщо для body записано таке правило:

```html
body { font-size: 14px}
```

![](html/copy-svgrepo-com.svg)

А для заголовка першого рівня таке:

```html
h1 { font-size: 3em }
```

![](html/copy-svgrepo-com.svg)

То браузер розрахує розмір заголовка в пікселях, відштовхуючись від розміру шрифту його батька (тобто body), а саме:

```html
3*14px = 42px
```

![](html/copy-svgrepo-com.svg)

Якщо для h1 буде записано правило:

```html
h1 { font-size: 300% }
```

![](html/copy-svgrepo-com.svg)

То розрахунок буде аналогічним:

```html
14px*300%/100% = 14px*3 = 42px
```

![](html/copy-svgrepo-com.svg)

Тобто, і em, % відштовхуються від розміру шрифту батьківського елемента. Майте на увазі, що шрифтові властивості успадковуються почергово. Тобто, якщо h1 знаходиться в <div class="logo">, наприклад, а не безпосередньо в body, і для <div class="logo"> не заданий розмір шрифту, то для div-а шрифт теж становитиме 14px, а заголовок буде розрахований за тією самою схемою. А ось, якщо буде встановлено правило:

```html
.logo { font-size: 1.3em }
```

![](html/copy-svgrepo-com.svg)

То шрифт заголовка буде наступного розміру:

```html
14px (від body)*1.3 (для .logo) *3 (для h1) = 54.6px
```

![](html/copy-svgrepo-com.svg)

І такий самий розрахунок у %, лише потрібно ще поділити на 100.

Одиниця виміру 1rem (_root em_) вказує на розмір шрифту, який дорівнює розміру, вказаному для шрифту кореневого елемента розмітки — тега <html>. Цей розмір встановлюється в браузері користувача (зазвичай це 16px); або задається для HTML-сторінки. І всі розрахунки здійснюються відносно цього розміру.

Ще одна відносна одиниця: 1ex — відштовхується від висоти символу x шрифту елемента в нижньому регістрі. Але за наявності більш зручної одиниці em, на практиці використовується дуже рідко.

Наразі використовують такі одиниці, як 1vw (_1% viewport width_) та 1vh (_1% viewport height_), розраховуються відносно щодо розмірів області перегляду — ширини або висоти відповідно. Використовують їх частіше не для вказівки розмірів шрифту, а для встановлення таких CSS-властивостей, як width і height, про які ми поговоримо на наступних заняттях. Також можна використовувати одиниці vmin або vmax, які позначають менше або більше значення vw і vh.
